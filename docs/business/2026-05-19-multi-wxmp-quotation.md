# 多微信小程序对接开发报价方案

日期：2026-05-19

## 一、需求背景

客户希望在同一租户下支持对接多个微信小程序，各小程序的主体功能与现有移动端微信小程序保持一致，仅对以下内容做配置化和差异化处理：

1. 移动端首页顶部 Banner 图片支持后台配置。
2. 微信小程序端（PACK 厂或 OEM）登录/注册隐藏手机号和邮箱注册入口，仅允许微信用户体系登录注册。
3. 微信小程序“我的”页面中的“关于我们”、隐私协议、用户政策支持按小程序单独配置。

基于当前系统现状，乙方提供两种开发方案供客户选择。

## 二、方案一：基于 PACK 厂的小程序配置方案

### 2.1 方案概述

该方案以现有机构用户中的 PACK 厂层级为配置入口，不新增 OEM 客户管理模块，也不调整现有电池生命周期层级。后台仅在 PACK 厂信息或相关管理页面中增加“微信小程序配置”，用于维护该 PACK 厂对应的小程序接入信息、首页 Banner、关于我们、隐私协议、用户政策等内容。

该方案开发量较小，适合客户当前仅需要快速支持多个 PACK 厂小程序差异化展示和微信登录的场景。

### 2.2 主要建设内容

| 序号 | 模块 | 工作内容 | 金额 |
| --- | --- | --- | ---: |
| 1 | 后端小程序配置 | 新增 PACK 厂维度的小程序配置数据结构、配置保存/查询接口，支持 AppID、AppSecret 等基础接入信息维护 | 1,200 元 |
| 2 | 后台管理页面 | 在现有 PACK 厂管理或机构管理入口中增加微信小程序配置表单，支持配置查看、编辑、启停等基础操作 | 1,000 元 |
| 3 | 首页 Banner 配置 | 增加小程序首页顶部 Banner 配置能力，移动端进入小程序后按对应配置展示；未配置时保留现有默认图 | 900 元 |
| 4 | 微信登录注册调整 | 微信小程序端隐藏手机号、邮箱注册入口，仅保留微信用户体系登录/注册流程 | 900 元 |
| 5 | 关于我们与协议配置 | 支持按 PACK 厂小程序单独配置“关于我们”“隐私协议”“用户政策”，移动端按当前小程序读取展示 | 1,000 元 |
| 6 | 联调、测试与文档 | 完成后台、后端、移动端联调，覆盖主要登录、配置读取、内容展示路径，并整理交付说明 | 1,000 元 |
|  | **合计** |  | **6,000 元** |

### 2.3 交付周期

预计开发周期：2 周。

| 阶段 | 时间 | 交付内容 |
| --- | --- | --- |
| 第 1 周 | 需求确认、数据结构与接口开发、后台配置页面开发 | 完成后端配置能力和 WEB 管理入口 |
| 第 2 周 | 移动端适配、登录注册调整、内容页面联调、测试修复 | 完成小程序端配置读取和整体交付测试 |

### 2.4 适用场景

- 当前客户主要以 PACK 厂为小程序运营主体。
- 不需要新增 OEM 客户层级。
- 不需要调整电池出厂、流转、生命周期管理链路。
- 希望以较低成本、较短周期完成多小程序接入。

### 2.5 方案边界

本方案不包含以下内容：

- 不新增 OEM 客户管理模块。
- 不新增 OEM 类型的 WEB 端账号、菜单、权限、数据范围。
- 不调整电池生命周期中的出厂对象和流转层级。
- 不处理 OEM 与 PACK 厂之间的上下级关系维护。

## 三、方案二：新增 OEM 客户管理与小程序配置方案

### 3.1 方案概述

该方案在后台管理的 BMS 管理模块下新增“OEM 客户管理”，完整支持 OEM 客户信息的新增、编辑、删除、查询，并为每个 OEM 客户配置独立的微信小程序接入信息、首页 Banner、关于我们、隐私协议、用户政策等内容。

同时，该方案会在现有总厂家与 PACK 厂之间引入 OEM 客户层级，电池生命周期管理需要支持“出厂到 OEM 客户”这一操作，WEB 端管理权限也需要新增 OEM 客户类型及对应的数据范围控制。

该方案开发量较大，但系统结构更完整，适合客户后续需要按 OEM 客户进行独立运营、授权、数据隔离和小程序管理的场景。

### 3.2 主要建设内容

| 序号 | 模块 | 工作内容 | 金额 |
| --- | --- | --- | ---: |
| 1 | OEM 客户数据模型 | 新增 OEM 客户基础数据结构、数据库迁移、组织层级关系设计，支持总厂家、OEM、PACK 厂之间的关系表达 | 1,600 元 |
| 2 | OEM 客户管理页面 | 在 WEB 后台 BMS 管理下新增 OEM 客户管理模块，支持 OEM 客户新增、编辑、删除、查询、详情查看等功能 | 1,800 元 |
| 3 | OEM 小程序配置 | 支持 OEM 维度配置微信小程序接入信息，包括 AppID、AppSecret、启停状态、关联租户/客户等信息 | 1,200 元 |
| 4 | 首页 Banner 配置 | 支持 OEM 小程序首页 Banner 独立配置，移动端按当前小程序/OEM 配置展示，未配置时保留默认兜底 | 900 元 |
| 5 | 微信登录注册调整 | 微信小程序端按 OEM 小程序进入后，隐藏手机号和邮箱注册入口，仅允许微信用户体系登录/注册 | 900 元 |
| 6 | 关于我们与协议配置 | 支持 OEM 小程序独立配置“关于我们”“隐私协议”“用户政策”，并在移动端按当前小程序展示 | 1,000 元 |
| 7 | 电池生命周期调整 | 电池出厂流程新增“出厂到 OEM 客户”操作，调整电池列表、详情、操作记录及相关状态流转展示 | 1,600 元 |
| 8 | WEB 权限与数据范围 | 新增 OEM 客户类型相关菜单、按钮、接口权限和数据范围控制，确保 OEM 账号仅访问授权数据 | 1,300 元 |
| 9 | 联调、测试与文档 | 覆盖 OEM 管理、小程序配置、微信登录、协议展示、电池出厂到 OEM、权限隔离等关键路径测试，并整理交付说明 | 1,700 元 |
|  | **合计** |  | **12,000 元** |

### 3.3 交付周期

预计开发周期：3 到 4 周。

| 阶段 | 时间 | 交付内容 |
| --- | --- | --- |
| 第 1 周 | 需求确认、OEM 数据结构设计、后端基础接口开发 | 完成 OEM 客户模型、基础接口和数据库迁移 |
| 第 2 周 | WEB 管理页面、OEM 小程序配置、内容配置开发 | 完成后台 OEM 管理和小程序配置能力 |
| 第 3 周 | 电池生命周期、权限数据范围、移动端适配 | 完成出厂到 OEM、权限隔离、小程序端展示和登录调整 |
| 第 4 周 | 联调测试、问题修复、验收支持 | 完成整体回归、交付文档和验收支持 |

### 3.4 适用场景

- 客户需要把 OEM 客户作为独立管理对象。
- 后续需要按 OEM 客户独立运营微信小程序。
- 电池业务链路需要体现“总厂家 -> OEM 客户 -> PACK 厂”的层级关系。
- WEB 后台需要支持 OEM 客户类型的权限、菜单和数据范围。
- 客户希望为后续 OEM 业务扩展预留完整系统能力。

### 3.5 方案边界

本方案包含 OEM 客户管理、小程序配置、电池出厂到 OEM、WEB 权限适配等核心能力，但不包含以下内容：

- 不包含微信开放平台主体认证、微信小程序注册、微信支付等第三方平台代办服务。
- 不包含客户侧小程序代码审核被微信平台驳回后的非系统原因整改。
- 不包含历史存量电池复杂归属数据的大规模人工清洗；如需批量迁移历史归属，可另行评估。
- 不包含除“关于我们、隐私协议、用户政策、首页 Banner”之外的更多页面内容运营系统。

## 四、两种方案对比

| 对比项 | 方案一：PACK 厂配置 | 方案二：OEM 客户管理 |
| --- | --- | --- |
| 报价 | 6,000 元 | 12,000 元 |
| 周期 | 2 周 | 3 到 4 周 |
| 改动范围 | 小 | 中到大 |
| 后台新增模块 | PACK 厂小程序配置 | OEM 客户管理 + OEM 小程序配置 |
| 是否新增组织层级 | 否 | 是，新增 OEM 客户层级 |
| 是否调整电池生命周期 | 否 | 是，支持出厂到 OEM 客户 |
| 是否新增 WEB 权限类型 | 少量配置权限 | 新增 OEM 客户类型权限和数据范围 |
| 后续扩展能力 | 适合短期快速接入 | 适合长期 OEM 独立运营 |
| 主要风险 | 后续如再引入 OEM，可能需要二次改造 | 开发范围更大，需更充分联调和验收 |

## 五、乙方建议

如果客户当前目标是快速支持多个 PACK 厂小程序上线，并且短期内不需要把 OEM 客户纳入完整业务链路，建议选择方案一，成本低、周期短、对现有系统影响较小。

如果客户后续明确需要发展 OEM 客户体系，并希望系统层面完整支持 OEM 客户管理、OEM 小程序配置、电池出厂到 OEM、OEM 数据权限隔离，则建议选择方案二。该方案初期投入更高，但后续扩展能力更完整，能减少未来重复改造。
