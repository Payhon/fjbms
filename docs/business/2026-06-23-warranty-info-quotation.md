# 移动端用户质保信息功能开发报价方案

日期：2026-06-23

## 一、需求背景

客户希望在富嘉云平台移动端新增用户质保信息能力：用户初次绑定电池设备后，可填写自己的质保联系人信息，包括姓名、联系电话，便于后续维修、保养和售后服务跟进。考虑 APP 存在国外使用场景，移动端质保信息仅做引导填写，不作为用户继续使用 APP 或小程序的强制条件。

同时，WEB 后台需要在 BMS 管理的电池详情页补充质保信息查看能力，方便管理人员在查看单个电池时同步了解该电池绑定用户、用户填写的质保信息、激活日期和质保截止日期。云平台端在电池激活后自动计算质保到期时间，并支持后台针对单台电池单独修改质保时长，以满足多设备质保年限不统一的业务情况。

本次开发目标是在现有移动端和后台管理基础上补充质保信息闭环，并在现有电池激活逻辑上补充质保到期日期计算与单台设备质保时长维护能力。

## 三、功能说明

### 3.1 移动端功能

1. 在“我的”页面新增“质保信息”菜单入口。
2. 用户进入质保信息页面后，可查看本人质保基本信息：
   - 姓名
   - 联系电话
3. 如果用户未填写质保基本信息，页面展示引导填写入口，但不强制填写，不影响继续使用 APP 或小程序其他功能。
4. 用户可填写、跳过或后续更新本人质保信息。
5. 页面展示当前用户关联的电池质保卡片，卡片内容包括：
   - 电池序列号
   - 激活日期
   - 质保截止日期
6. 无质保信息、无绑定电池或数据缺失时，提供合理空状态提示。

### 3.2 云平台质保计算与维护

1. 电池激活后，云平台根据激活日期自动计算质保截止日期。
2. 后台支持针对单台电池单独修改质保时长。
3. 系统支持不同设备使用不同质保年限，不要求所有设备质保年限统一。
4. 修改单台电池质保时长后，该电池质保截止日期同步更新，并在移动端质保卡片和后台电池详情中展示。

### 3.3 WEB 后台功能

1. 在 “BMS 管理 > 电池列表 > 电池详情” 页面新增“质保”Tab。
2. “质保”Tab 展示该电池关联绑定用户信息，包括：
   - 绑定用户基本信息
   - 用户填写的质保姓名
   - 用户填写的质保联系电话
   - 电池激活日期
   - 电池质保截止日期
   - 当前质保时长
3. 支持后台人员单独调整当前电池的质保时长，用于处理不同设备质保年限不统一的情况。
4. 若电池未绑定用户或用户未填写质保信息，则展示空状态或“未填写”状态，方便后台人员识别。

本次交付包含：

1. 移动端“我的 > 质保信息”入口和页面。
2. 用户质保基本信息填写、保存和回显能力；移动端仅引导填写，不强制填写。

4. 电池激活后的质保截止日期自动计算能力。

7. WEB 后台单台电池质保时长修改能力，支持多设备质保年限不统一。
8. 基础联调测试和交付说明。

## 六、范围边界

本次报价不包含以下内容：

1. 不包含复杂售后工单、维修工单、保养工单流程。
2. 不包含质保规则的复杂批量配置化，例如按不同客户、型号、批次自动套用不同质保年限；本次仅包含电池激活后默认计算质保截止日期，以及后台对单台电池单独修改质保时长。
3. 不包含历史存量电池质保信息的大规模人工清洗或批量导入。
4. 不包含短信、电话外呼、公众号通知等售后触达能力。
5. 不包含新的后台统计报表、质保到期提醒列表或质保运营看板。

如客户后续需要扩展售后工单、按客户/型号/批次的质保规则配置、到期提醒、批量导入或统计报表，可在本次功能基础上另行评估。
