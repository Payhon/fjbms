# 移动端用户质保信息功能开发报价方案

日期：2026-06-23

## 一、需求背景

客户希望在富嘉云平台移动端新增用户质保信息能力：用户初次绑定电池设备后，可填写自己的质保联系人信息，包括姓名、联系电话，便于后续维修、保养和售后服务跟进。

同时，WEB 后台需要在 BMS 管理的电池详情页补充质保信息查看能力，方便管理人员在查看单个电池时同步了解该电池绑定用户、用户填写的质保信息、激活日期和质保截止日期。

本次开发目标是在现有移动端和后台管理基础上补充质保信息闭环，不改变现有电池绑定、激活和质保计算主流程。

## 三、功能说明

### 3.1 移动端功能

1. 在“我的”页面新增“质保信息”菜单入口。
2. 用户进入质保信息页面后，可查看本人质保基本信息：
   - 姓名
   - 联系电话
3. 如果用户未填写质保基本信息，页面展示引导填写入口。
4. 用户可填写或更新本人质保信息。
5. 页面展示当前用户关联的电池质保卡片，卡片内容包括：
   - 电池序列号
   - 激活日期
   - 质保截止日期
6. 无质保信息、无绑定电池或数据缺失时，提供合理空状态提示。

### 3.2 WEB 后台功能

1. 在 “BMS 管理 > 电池列表 > 电池详情” 页面新增“质保”Tab。
2. “质保”Tab 展示该电池关联绑定用户信息，包括：
   - 绑定用户基本信息
   - 用户填写的质保姓名
   - 用户填写的质保联系电话
   - 电池激活日期
   - 电池质保截止日期
3. 若电池未绑定用户或用户未填写质保信息，则展示空状态或“未填写”状态，方便后台人员识别。

## 六、范围边界

本次报价不包含以下内容：

1. 不包含复杂售后工单、维修工单、保养工单流程。
2. 不包含质保规则的复杂配置化，例如按不同客户、型号、批次配置不同质保年限；本次默认复用现有电池质保数据或现有系统计算结果。
3. 不包含历史存量电池质保信息的大规模人工清洗或批量导入。
4. 不包含短信、电话外呼、公众号通知等售后触达能力。
5. 不包含新的后台统计报表、质保到期提醒列表或质保运营看板。

如客户后续需要扩展售后工单、质保规则配置、到期提醒、批量导入或统计报表，可在本次功能基础上另行评估。
