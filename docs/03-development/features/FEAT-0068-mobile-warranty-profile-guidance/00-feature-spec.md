# FEAT-0068 移动端质保资料填写引导 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-07-10
- related_feature: FEAT-0068
- version: v0.1.0

## 1. 背景与目标
- FEAT-0062 已提供终端用户质保联系人资料页，但用户绑定设备后不一定会主动填写，厂家无法稳定取得售后联系人。
- 目标是在不阻断设备添加的前提下，引导终端用户完善姓名和联系电话，并在后续启动时以低打扰方式持续提示。

## 2. 范围
### In Scope
- 当前用户真实新增绑定设备后，未完成质保资料时弹出“立即填写 / 以后再说”引导。
- 登录态 App 启动或回到前台时静默检查；需要补充资料时在“我的”Tab 与“我的 > 质保信息”入口显示红点。
- 质保页顶部显示售后资料提示；保存完整资料后同步清除所有提醒。
- 扩展既有 App 质保 profile 和设备绑定接口响应，补齐中英文文案、测试、文档和看板。

### Out of Scope
- 强制填写、阻断设备绑定或设备详情访问。
- 短信、电话、推送等外部提醒。
- SQL 表结构变更、售后工单和质保到期提醒。

## 3. 验收标准
1. 终端用户本次新增绑定设备且质保资料未完成时，显示可取消的引导对话框；“立即填写”进入可编辑质保资料页，“以后再说”继续原设备详情流程。
2. 重复扫描、已绑定设备和机构用户不触发该对话框。
3. 质保资料完成仅以 `user_warranty_infos` 中用户主动保存的非空姓名和联系电话判定，不把账号资料回填误判为完成。
4. 有绑定设备但资料未完成的终端用户，每次启动或回到前台后只做静默检查并显示红点，不重复弹窗。
5. 保存完整联系人资料后，原生 TabBar、微信自定义 TabBar、我的页菜单红点和资料页提示同时消失。
6. `user_warranty_infos` 没有当前用户记录时，质保页必须明确显示“个人质保信息未填写”和填写入口；不得展示账号姓名或手机号作为个人质保资料。

## 4. 风险与约束
- 既有 profile 保存接口仍允许保存不完整资料；不完整资料继续保留提醒。
- 网络异常时不阻断绑定或页面跳转，并在下次前台检查时重试。
- 微信小程序使用自定义 TabBar，须与 App 原生 TabBar 分别同步红点。

## 5. 回滚方案
- 回滚 UniApp 的引导、Pinia 状态和红点展示后，设备绑定和质保资料维护仍维持原有行为。
- 回滚后端新增响应字段后，旧客户端可继续兼容原接口；无数据库回滚项。
