# FEAT-0068 移动端质保资料填写引导 - 发布说明

- status: review
- owner: payhon
- last_updated: 2026-07-10
- related_feature: FEAT-0068
- version: v0.1.0

## 变更内容
- 终端用户首次新增绑定设备后，在未完成质保联系人资料时会获得可跳过的填写引导。
- App 启动或前台恢复后，未完成资料的已绑定用户会看到“我的”Tab 与质保菜单红点，并可在质保页顶部看到售后资料提示。
- 姓名和联系电话完整保存后，所有引导自动清除。
- 个人质保信息展示只读取 `user_warranty_infos`；无记录时明确显示“个人质保信息未填写”，不再展示账号姓名或手机号。

## 发布步骤
1. 发布 backend 服务，无 SQL 迁移。
2. 构建并发布 `fjbms-uniapp`。
3. 使用终端用户账号绑定新设备，分别验证“立即填写”和“以后再说”。
4. 验证微信小程序、iOS、Android 前台启动后的红点和资料保存清理行为。

## 回滚说明
- 回滚 backend 与 UniApp 代码即可停止新提醒；新增响应字段可被旧客户端忽略。
- 无数据库结构或历史数据回滚项。
