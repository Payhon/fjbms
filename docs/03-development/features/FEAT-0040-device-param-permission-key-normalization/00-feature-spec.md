# FEAT-0040 设备参数权限 Key 归一化与移动端参数显隐修复 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-15
- related_feature: FEAT-0040
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - 后台“系统管理 > 权限管理 > 设备参数权限”中的部分节点仍在使用历史遗留的非标准 key，例如 `403:feedback_delay`、`410:alarm_release`。
  - 移动端设备详情参数页按协议寄存器地址判断参数权限，例如 `403`、`410`、`42a`。
  - 当前前后端 key 口径不一致，导致部分参数在后台取消勾选后，移动端仍可能继续显示。
- 目标：
  1. 统一设备参数权限 key 为纯寄存器地址口径。
  2. 保持对历史已保存旧 key 的兼容读取，避免要求租户手工重配权限。
  3. 修复移动端“单体设置 / 总压设置 / 电流设置 / 温度设置”中参数显隐异常。

## 2. 范围
### In Scope
- 修正后端设备参数权限树中非标准 key。
- 对历史 `device_param_permissions` 做运行时归一化兼容。
- 修正后台权限管理页读取/保存时返回的设备参数权限 key。
- 补充单测验证 key 归一化与权限树唯一性。
- 文档和看板同步更新。

### Out of Scope
- 不修改移动端参数页 UI 布局。
- 不改动设备协议读写实现。
- 不新增数据库字段。

## 3. 用户价值
- 后台取消勾选设备参数权限后，移动端显隐结果与配置保持一致。
- 租户无需手工清理历史权限数据即可获得修复结果。

## 4. 验收标准
1. 后台设备参数权限树的叶子节点 value 不再使用 `403:feedback_delay` 这类历史 key。
2. 历史保存的旧 key 在当前接口返回和移动端权限判断中会自动归一化到纯地址 key。
3. 取消以下参数权限后，移动端对应参数不再显示：
   - 单体回馈过充保护延时
   - 单体过放告警解除电压
   - 单体过放保护解除电压
   - 单体过放告警解除延时
   - 单体过放保护解除延时
   - 充电过流保护小电流延时
   - 放电过流告警延时
   - 放电过流小电流保护延时
   - 放电过流大电流保护延时
   - MOS高温保护延时
   - MOS高温保护解除延时
   - 放电高温保护解除温度
4. 新增单测通过，且不会引入后端编译错误。

## 5. 风险与约束
- 设备参数权限当前按寄存器地址授权，而不是按高低字节分别授权；同寄存器高低字节参数需要共享同一权限 key。
- 历史租户数据中可能同时存在新旧 key，归一化时必须去重并保持兼容。
