# FEAT-0040 设备参数权限 Key 归一化与移动端参数显隐修复 - 实施日志

- last_updated: 2026-04-15
- related_feature: FEAT-0040
- version: v0.1.0

## 1. 实施记录
1. 对比移动端 `getParamPermissionKey()` 与后端设备参数权限树节点 value，确认双方口径不一致。
2. 定位到后端权限树中共有 12 个参数节点仍在使用带后缀的历史 key。
3. 确认问题集中出现在以下地址：
   - `403`
   - `40e`
   - `40f`
   - `410`
   - `423`
   - `42a`
   - `42b`
   - `43a`
   - `43b`
   - `44b`
4. 已将上述节点统一改为纯地址 key，并对 `410`、`42a` 这类同地址双参数节点做合并。
5. 已在后端权限服务中补充旧 key 归一化映射，并在列表读取、保存、当前用户权限查询三条链路统一接入。
6. 已修复归一化逻辑对 `factory:*` / `function:*` 高级参数权限的误伤问题：
   - 仅对寄存器地址类 key 做小写标准化
   - 对 `factory:*` / `function:*` 做大小写不敏感的规范 key 恢复
   - 兼容已被错误写入小写形式的历史权限值
7. 已补充后端单测覆盖：
   - 旧 key 归一化
   - `factory:*` / `function:*` 规范 key 恢复
   - 权限树 key 唯一性与规范性
