# FEAT-0017 BMS 状态寄存器地址调整、移动端保护状态卡片与旧板兼容读取 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-30
- related_feature: FEAT-0017
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - 最新协议文档将 `0x12D~0x135` 区间重新定义，其中 `0x12D~0x12E` 改为“总放容量”，保护状态从 `0x12D` 后移到 `0x12F~0x130`。
  - 当前项目后端状态解析器、UniApp 协议解析器以及移动端设备详情仪表盘仍按旧地址读取保护状态，导致保护状态显示错位。
  - 当前移动端仪表盘对保护状态采用“有激活项才显示标签”的呈现方式，无法查看完整保护位开关状态。
  - 旧款 BMS 板在 `0x135~0x139` 寄存器没有实现，UniApp 蓝牙设备详情若按 `0x100` 开始连续读取整段状态，轮询阶段会因为跨过未实现地址而失败。
  - 设备详情顶部状态入口必须区分 `0x1xx` 只读状态与 `0x4xx` 读写配置项：配置项开启不代表当前故障、告警或保护已经触发。
- 目标：
  1. 同步项目内状态寄存器解析和相关配置项，使其与最新协议一致。
  2. 为状态模型补充“总放容量”字段，避免 `0x12D~0x12E` 区间语义缺失。
  3. 将 UniApp 设备详情仪表盘的保护状态改为独立可折叠卡片，内部列表固定展示各保护项及开关文本。
  4. 将 UniApp 设备详情仪表盘温度区域改为根据 `0x100(L)` 的电芯温度数量动态展示 `T1~Tn`。
  5. 将 UniApp 蓝牙设备详情状态读取改为“两段读取 + 缺口回填”，兼容旧款 BMS 板未实现 `0x135~0x139` 的情况。
  6. 将移动端与 Web 端设备详情仪表页的故障、告警、保护入口统一改为按只读状态位判断。

## 2. 范围
### In Scope
- 更新 `docs/02-architecture/system-design/protocol-source/device-comm-basic.md` 中 `0x12D~0x135` 的主维护文档描述。
- 更新后端 BMS 状态解析、状态扁平化以及模拟器寄存器布局。
- 更新 Frontend Web BMS 协议状态类型、参数注册和设备详情 BMS 面板展示。
- 更新 UniApp 协议状态类型、参数注册和状态解析地址。
- 更新 UniApp 设备详情仪表盘的保护状态展示结构与中英文文案。
- 更新 UniApp 设备详情仪表盘温度区域，使其按电芯温度数量渲染多行温度。
- 更新 UniApp 与 Frontend Web `readAllStatus()` 读取策略：第一次读取 `0x100~0x134` 共 `0x35`（53）个寄存器，单独尝试读取 `0x135` 告警高 16 位，第二次从 `0x141` 开始读取到动态状态区末尾，并对旧板不支持的缺口进行兼容回填。

### Out of Scope
- 不使用 `0x4xx` 读写配置项参与故障、告警或保护入口判断。
- 不改动 Web 管理端的设备详情或图表展示。
- 不补充失效状态 `0x131` 的独立详情卡片，仅用于顶部“故障”入口判断与弹层列表。

## 3. 验收标准
1. 后端和 UniApp 读取保护状态时均从 `0x12F~0x130` 解析。
2. 后端、Frontend Web 与 UniApp 的告警状态均从 `0x134~0x135` 解析。
3. 状态模型中可读取 `0x12D~0x12E` 对应的总放容量字段。
4. UniApp 设备详情仪表盘中：
   - 保护状态独立成卡片；
   - 卡片默认收起，点击后展开/收起；
   - 内部列表展示所有保护项；
   - 每一项使用文本显示“开启/关闭”或对应英文状态。
5. UniApp 设备详情仪表盘温度区中：
   - 固定展示 `MOS温度` 与 `环境温度`；
   - 按 `0x100(L)` 电芯温度数量展示 `T1` 起的多条电芯温度；
   - 当电芯温度数量为 `0` 时，不展示电芯温度行。
6. 移动端与 Web 端设备详情仪表页顶部状态入口：
   - “故障”仅按 `0x131` 失效状态位判断；
   - “告警”仅按 `0x134~0x135` 告警状态位判断；
   - “保护”仅按 `0x12F~0x130` 保护状态位判断；
   - `0x4xx` 配置项开启不触发任何顶部状态入口。
7. 当没有任何激活保护项时，保护状态卡片仍然显示，只是列表状态均为关闭。
8. UniApp 与 Frontend Web 读取全量状态时：
   - 第一次读取范围固定为 `0x100~0x134`，数量为 `0x35`（53）个寄存器；
   - 支持单独读取 `0x135`，用于补齐 `0x134~0x135` 告警状态 32BIT；
   - 第二次读取从 `0x141` 开始；
   - `0x135` 读取失败或 `0x136~0x140` 缺口不会导致旧款 BMS 轮询失败；
   - `alarmStatus` 中依赖 `0x135` 的高位部分在旧板兼容读取失败时按默认值回填。

## 4. 风险与约束
- 若后端与 UniApp 仅修改其中一侧，会造成蓝牙直连与云端状态口径不一致。
- 若模拟器寄存器布局不随协议同步，联调环境会继续产生假阳性或假阴性的保护状态结果。
- 当前状态位映射必须与协议描述保持一致，若后续再次调整位定义，需要同时检查移动端、Web 端和后端解析表。
- 旧板兼容读取必须避免因 `0x135` 或 `0x136~0x140` 未实现而中断轮询；因此 `0x135` 采用可选单寄存器读取，失败时在内存拼装阶段以默认值回填。

## 5. 回滚方案
- 回滚 FEAT-0017 的状态解析、文档与 UniApp 页面改动，恢复旧寄存器地址读取方式和旧版保护状态标签展示。
- 回滚时同时撤销新增的“总放容量”状态字段与 UniApp 两段读取兼容逻辑，避免协议文档与运行时模型不一致。
