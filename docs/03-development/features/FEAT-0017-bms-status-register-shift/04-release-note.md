# FEAT-0017 BMS 状态寄存器地址调整、移动端保护状态卡片与旧板兼容读取 - 发布说明

- last_updated: 2026-04-30

- version: v0.1.0

## 发布内容
- 同步最新 BMS 协议中 `0x12D~0x135` 状态寄存器定义。
- 修正保护状态寄存器读取地址为 `0x12F~0x130`。
- 明确设备详情仪表页顶部状态入口来源：
  - 故障仅来自 `0x131` 失效状态；
  - 告警仅来自 `0x134~0x135` 告警状态；
  - 保护仅来自 `0x12F~0x130` 保护状态；
  - `0x4xx` 读写配置项不参与顶部状态入口判断。
- 新增总放容量状态字段 `totalDischargeCapacityRaw`。
- 新增失效状态字段 `failureStatus`，用于故障入口与状态事件输出。
- Web 端 BMS 面板新增故障、告警、保护顶部入口和弹层列表，展示口径与移动端一致。
- Web 端 BMS 面板的保护状态改为可展开全量列表，并以文本显示各保护项开关状态。
- 移动端设备详情仪表盘新增独立保护状态折叠卡片，并以文本显示各保护项开关状态。
- UniApp 蓝牙设备详情全量状态读取改为两段读取：
  - 第一段读取 `0x100~0x134`，数量为 `0x35`（53）个寄存器；
  - 单独尝试读取 `0x135`，用于补齐告警状态高 16 位；
  - 第二段从 `0x141` 读取到动态状态区末尾；
  - `0x136~0x140` 作为旧板兼容缺口在内存中回填默认值，旧板不支持 `0x135` 时该寄存器也按默认值处理。

## 影响范围
- `backend/` BMS 状态解析与桥接状态输出。
- `frontend/` Web BMS 协议解析与设备详情 BMS 面板展示。
- `fjbms-uniapp/` 蓝牙/MQTT 状态解析与设备详情仪表盘展示。
- `docs/` 协议主维护文档与项目看板。

## 发布与回滚提示
- 发布前建议执行后端定向 `go test`、Frontend `pnpm typecheck` 与 UniApp `tsc --noEmit`。
- 如需回滚，需同时回滚协议解析地址、状态模型字段、UniApp 仪表盘展示以及 UniApp 两段读取兼容逻辑，保持协议口径一致。
