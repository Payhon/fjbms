# FEAT-0038 后台上传链路修复与云直传接入 - 发布说明

- status: review
- owner: payhon
- last_updated: 2026-04-09
- related_feature: FEAT-0038
- version: v0.1.0

## 1. 发布内容
- 后端上传文件类型校验新增 `appPackage` 与 `wgtPackage` 支持。
- 后台共用上传器在云存储启用时优先走云直传链路，不再默认经业务服务器中转。
- 共用上传器新增组件内上传进度、阶段状态与错误反馈。
- 图片、应用安装包、升级发布安装包字段补齐 URL 回填语义。

## 2. 影响范围
- 影响模块：`backend/` 文件服务、`frontend/` 共用上传组件与应用管理页面。
- 不涉及数据库表结构变更。
- 不影响未接入共用上传器的旧上传页面。

## 3. 升级步骤
- 部署后端与前端最新代码。
- 若使用云直传，确认对象存储桶已放行浏览器上传所需跨域配置。

## 4. 回滚步骤
- 回滚共用上传器的 `custom-request` 直传逻辑与后端文件白名单调整。
- 恢复页面层安装包字段原有 `value-mode` 配置。
- 回滚 FEAT-0038 文档与看板状态。

## 5. 已知问题
- 云直传依赖对象存储环境配置，若 CORS 或签名策略不完整，浏览器会直接报错。
- 历史上已保存为内部路径的旧数据不会自动迁移为外链 URL。
