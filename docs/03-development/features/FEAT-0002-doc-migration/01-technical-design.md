## 1. 方案概览
采用“清单驱动迁移”：先产出统一映射表，再分批迁移并在看板追踪。

## 2. 迁移策略
1. 盘点：枚举 `doc/` 所有文件并分类（产品/架构/开发/运维/归档）。
2. 映射：为每份历史文档指定 `docs/` 目标路径和处理方式。
3. 执行：按 P0 -> P1 -> P2 分批迁移。
4. 验收：更新状态、补索引、记录风险与阻塞。

## 3. 命名与路径规则
- 目标路径遵循 `docs/00~05` 分层。
- 文件名统一 kebab-case（中文标题可保留在一级标题中）。
- 历史阶段性报告优先归档到 `docs/04-project-tracking/archive/`。

## 4. 安全与权限
- 不在迁移文档中写入密钥、密码或环境私密信息。
- 涉及外部接口文档时保留脱敏样例。

## 5. 测试策略
- 抽样检查：每批迁移后随机抽查至少 3 篇文档链接有效性。
- 结构检查：确保新增文档可从目录索引或看板追踪到。

## 6. 兼容性与迁移
- 迁移周期内保留 `doc/` 原文件，避免历史链接立即失效。
- 稳定后再追加“旧路径 -> 新路径”跳转说明。
