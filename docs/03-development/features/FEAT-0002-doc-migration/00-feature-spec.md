# FEAT-0002 历史文档迁移与分层重构 - 功能规格

## 1. 背景与目标
- 背景：当前历史文档集中在 `doc/`，与新治理目录 `docs/` 并存，入口分散。
- 目标：建立可执行迁移清单，按优先级把存量文档映射到新分层，避免知识丢失。

## 2. 范围
### In Scope
- 形成迁移清单与目标路径映射。
- 明确迁移优先级（P0/P1/P2）与处理策略（迁移/合并/归档）。
- 约束迁移过程中的命名、状态和验收标准。

### Out of Scope
- 大规模一次性重写所有历史文档内容。
- 改造与文档无关的代码实现。

## 3. 验收标准
1. `docs/04-project-tracking/doc-migration-checklist.md` 完成并可执行。
2. `board.md` 中存在 FEAT-0002 且状态与实际一致。
3. PR 模板包含文档治理检查项，确保后续迁移可持续。

## 4. 风险与约束
- 风险：历史文档重名或信息冲突导致重复维护。
- 风险：迁移后未建立跳转或索引，团队找不到新入口。
- 约束：迁移以非破坏方式推进，优先“复制+校验+归档”。

## 5. 回滚方案
- 若某批迁移引发引用错误，先回退该批次变更，保留 `doc/` 原文档不删除。
