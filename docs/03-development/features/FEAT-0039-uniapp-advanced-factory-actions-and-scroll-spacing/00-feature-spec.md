# FEAT-0039 UniApp 设备详情高级参数出厂命令补齐与底部遮挡修复 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-17
- related_feature: FEAT-0039
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - `fjbms-uniapp` 设备详情页“设置 > 高级参数 > 出厂配置”当前缺少协议文档已定义的部分工厂命令。
  - 高级参数弹层底部滚动空间不足，末尾工厂命令会被设备详情底部 Tab 挡住，导致无法完整滚动和点击。
- 目标：
  1. 按 `doc/oriigin/device_comm_protocol_write_v2.md` 补齐缺失工厂命令。
  2. 修复高级参数弹层底部可滚动空间不足的问题。
  3. 保证新增工厂命令可被设备参数权限体系配置和控制。

## 2. 范围
### In Scope
- `fjbms-uniapp/pages/device-battery/components/params-tab.vue` 工厂命令列表补齐。
- `fjbms-uniapp/lang/zh-CN.ts`、`fjbms-uniapp/lang/en-US.ts` 新增工厂命令文案。
- `backend/internal/service/device_param_permission_tree.go` 新增工厂命令权限节点。
- 高级参数弹层滚动区底部留白优化。
- FEAT-0039 文档和项目看板同步更新。

### Out of Scope
- 不改动工厂命令的协议寄存器地址与发送方式。
- 不调整设备详情底部一级 Tab 结构。
- 不新增新的后端接口或数据库变更。

## 3. 用户价值
- 工厂和运维角色可在移动端直接执行完整的协议定义工厂操作。
- 高级参数页面末尾菜单可稳定滚动到可点击区域，减少误以为功能缺失或失效的问题。

## 4. 验收标准
1. “出厂配置”中新增以下命令项，并点击后仍走现有确认与发送流程：
   - 擦除当前参数
   - 擦除历史记录
   - 擦除循环次数
   - 清除保护状态
   - 复位保护板
2. 新增命令的中英文文案显示正确。
3. 组织类型权限配置页可看到上述新增工厂命令权限项。
4. 高级参数弹层滚动到最底部时，末尾命令不会被设备详情底部 Tab 挡住，可正常点击。
5. 本次改动不会引入 UniApp TypeScript 编译错误，也不会引入后端编译错误。

## 5. 风险与约束
- `0x057A~0x057B` 的工厂命令依赖位掩码写入，新增命令必须与协议文档位定义保持一致。
- “复位保护板”当前协议口径已明确为 `BIT21 -> 0x00200000`，示例帧为 `7F 55 FE 00 10 05 7A 00 02 04 00 20 00 00 4E 92 FD`，相关实现和文档需保持一致。
- 滚动留白需兼容不同机型安全区和底部 Tab 高度，不能只按单一平台写死像素值。

## 6. 回滚方案
- 回滚 `params-tab.vue` 中新增工厂命令和底部留白逻辑。
- 回滚中英文文案和后端设备参数权限树的新增节点。
- 回滚 FEAT-0039 文档和看板状态。
