# FEAT-0009 管理端登录验证码与二维码配置展示 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-03-06
- related_feature: FEAT-0009
- version: v0.1.0

## 1. 背景与目标
- 背景：当前后台管理端登录仅账号密码校验，存在被爬虫或机器人持续暴力尝试密码的安全风险；同时登录页缺少便捷引导入口（微信小程序与 App 下载二维码）。
- 目标：
  1) 登录页新增图形验证码并参与后端校验。
  2) 系统设置 -> 主题设置新增“微信小程序二维码”“App 下载页二维码”配置项（本地上传图片）。
  3) 登录页在登录表单下方展示上述两张二维码，并保证排版美观。

## 2. 范围
### In Scope
- `backend`：新增登录验证码生成/校验能力；登录接口增加验证码参数校验；Logo/主题设置模型扩展二维码字段。
- `frontend`：密码登录模块新增验证码输入与刷新；主题设置页面新增二维码上传项；登录页新增二维码展示区块。
- 数据库：`logo` 表新增二维码字段，并兼容历史数据。

### Out of Scope
- 不改动 APP/小程序登录流程（`/app/auth/*`）。
- 不改动短信/邮箱验证码登录流程。
- 不新增第三方验证码服务接入（使用现有服务能力实现）。

## 3. 用户价值
- 降低管理端登录口令被自动化暴力破解的风险。
- 让用户在登录页即可快速扫码进入小程序或下载 App，减少跳转步骤。
- 管理员可在系统设置统一维护二维码图片，无需改代码。

## 4. 验收标准
1. 打开管理端登录页，用户名和密码输入框下方显示图形验证码输入与验证码图片，支持刷新。
2. 登录请求必须携带验证码 ID 和验证码文本；验证码错误或过期时，登录失败并提示。
3. 系统设置 -> 主题设置中可上传并保存：
   - 微信小程序二维码
   - App 下载页二维码
4. 登录页在登录表单下方展示已配置二维码（任一未配置时仅展示已配置项），布局在桌面与移动端均不错位。

## 5. 风险与约束
- 风险：验证码缓存依赖 Redis；若 Redis 异常会影响登录可用性。
- 风险：图片字段扩展涉及数据库迁移，需要确保线上迁移兼容历史行。
- 约束：保持现有登录 API 兼容性（新增字段但不破坏既有响应结构）。

## 6. 回滚方案
- 回滚 FEAT-0009 相关前后端提交。
- 数据库层保留新增字段不影响旧逻辑；若需彻底回退，可执行列删除 SQL（需评估线上数据影响）。
