# FEAT-0044 电池列表回退操作 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-21
- related_feature: FEAT-0044
- version: v0.1.0

## 1. 背景与目标
- 当前后台电池列表已有“调拨”能力，适用于 PACK 向下级机构下发电池。
- 当经销商或门店需要把电池退回给上一级来源机构时，现有调拨表单要求人工选目标机构，不符合“由谁调拨来的，就回退给谁”的业务规则。
- 目标：
  1. 在 `后台管理 > BMS 管理 > 电池列表` 新增独立“回退”操作。
  2. 回退目标由系统按最近一次入库来源自动反推，不允许前端手工选择。
  3. 回退成功后同步写入组织转移记录和电池操作日志。

## 2. 范围
### In Scope
- 后端新增回退预览接口与执行接口。
- 回退仅允许经销商/门店账号对自己当前持有的库存执行。
- 回退目标取最近一次 `device_org_transfers.to_org_id = 当前持有机构` 的来源组织。
- 前端电池列表新增“回退”菜单项与确认弹窗。
- 电池操作日志新增 `ROLLBACK` 类型。
- 同步权限 SQL、多语言与项目看板。

### Out of Scope
- 不改造现有“调拨”接口表单。
- 不新增批量回退。
- 不允许跨支路、跨租户或手工指定回退机构。

## 3. 验收标准
1. 经销商持有且来源为 PACK 的电池，可预览并成功回退到该 PACK。
2. 门店持有且来源为经销商的电池，可预览并成功回退到该经销商。
3. 当前账号不是电池当前持有机构时，不允许回退。
4. 没有可匹配来源记录、来源机构不存在或不在合法上级链路时，不允许回退，并返回明确原因。
5. 前端确认弹窗展示“当前机构”和“回退给谁”，不展示机构选择控件。
6. 回退成功后，`device_batteries.owner_org_id`、`device_org_transfers`、`battery_operation_logs` 同步更新。
7. 运营日志支持筛选并展示 `ROLLBACK`。

## 4. 风险与约束
- 预览与执行都必须复用同一套回退解析逻辑，避免“预览可回退、提交时报错”的规则漂移。
- 历史脏数据可能导致来源机构缺失或链路异常，此时必须直接拒绝，不做静默回退。
- 前端入口仅做体验收敛，最终以后端校验结果为准。

## 5. 回滚方案
- 回滚前端“回退”入口、确认弹窗与新 API 调用。
- 回滚后端回退预览/执行接口与 `ROLLBACK` 日志类型。
- 回滚权限补丁 `50.sql`，并按需移除 `org_type_permissions.ui_codes` 中新增的回退权限码。
