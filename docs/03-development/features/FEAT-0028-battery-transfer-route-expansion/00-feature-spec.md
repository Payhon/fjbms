# FEAT-0028 电池调拨路径扩展 - 功能规格

- status: archived
- owner: payhon
- last_updated: 2026-04-21
- related_feature: FEAT-0028
- version: v0.1.0

> 历史说明：本规格中的“经销商可回 PACK、门店可回经销商”矩阵已在 FEAT-0045 中被收口替代。当前生效口径请以 `FEAT-0045` 为准。

## 1. 背景与目标
当前后台管理 `BMS 管理 > 电池列表` 的调拨权限矩阵过窄，后端只允许 `PACK -> 经销商/门店`，并误将经销商限制成“只能门店间调拨”。结果是经销商无法调拨回 PACK、无法调拨至门店，门店也无法调拨回经销商。

本次需要把调拨路径调整为符合业务流转：
- 经销商可以调拨回 PACK 厂
- 经销商可以调拨至门店
- 门店可以调拨至经销商

## 2. 范围（In Scope / Out of Scope）
In Scope:
- 后端 `battery/transfer` 权限矩阵调整。
- 组织选项接口支持经销商选择上级 PACK、门店选择上级经销商。
- 管理端调拨弹窗目标机构类型按当前账号角色动态收敛。
- 文档与看板同步。

Out of Scope:
- 不改造“出厂”能力。
- 不新增跨租户或跨组织树调拨。
- 不扩展新的审批流、签收流或批量调拨页面。

## 3. 用户价值
- 经销商与门店可按实际业务流转回退或下发库存。
- 调拨权限与组织树关系一致，减少“有入口但必失败”的体验问题。
- 保持电池生命周期记录完整，便于追踪。

## 4. 验收标准
- 经销商持有的电池可调拨到其上级 PACK 厂。
- 经销商持有的电池可调拨到其下级门店。
- 门店持有的电池可调拨到其上级经销商。
- 非法目标仍被拒绝，例如门店不能直接调拨到 PACK。
- 管理端弹窗只展示当前账号可选的目标机构类型。

## 5. 风险与约束
- 必须严格限制在当前组织树链路内，不能允许跨支路调拨。
- 前端选项收敛仅改善体验，最终仍以后端权限校验为准。
- 组织选项接口被多个页面复用，扩展上级选项逻辑时需避免破坏现有筛选场景。

## 6. 回滚方案
- 回滚后端调拨权限矩阵与组织选项扩展。
- 回滚前端调拨弹窗目标机构类型收敛逻辑。
- 保留历史转移记录，不做数据回退。
