# FEAT-0031 App 上架准备与公共页面完善 - 发布说明

- status: in_progress
- owner: payhon
- last_updated: 2026-04-15
- related_feature: FEAT-0031
- version: v0.1.0

## 1. 发布内容
- 新增 App 公开隐私政策页、用户政策页和发行下载页。
- App 管理表单补齐发行信息维护能力。
- 升级中心支持更适合上架前发布的升级包填写方式。
- UniApp 登录、注册设密页改为公网 H5 协议页入口。
- UniApp 登录页底部“其他登录方式”按运行环境动态显示，非微信环境不再展示空标题。
- UniApp 升级检测接口新增定向控制台日志，便于排查版本检查请求与响应。
- 升级检测继续沿用现有自动检查与弹窗流程。
- UniApp iOS 权限文案补齐为明确用途描述，满足 App Store 审核对相机 / 蓝牙权限说明的要求。
- UniApp 主扫码入口统一收敛为仅摄像头扫码，规避 iPad 审核点击“相册”时的 H5+ 异常。
- 新增 `fjbms-uniapp/scripts/check_ios_ipa.sh`，用于送审前校验 `ipa` 的权限文案、追踪声明、图标资源与版本号。
- UniApp App 名称改为打包时国际化配置：简体中文 / 繁体中文显示 `富嘉BMS`，其他已覆盖语言显示 `FUJIA BMS`。

## 2. 影响范围
- `frontend/` 公共路由与 App 管理 / 升级中心页面。
- `backend/` 公共内容与公开 App 信息接口。
- `fjbms-uniapp/` 登录、注册设密、扫码入口、iOS 打包隐私描述与送审核验链路。
- `fjbms-uniapp/` 的 App 名称国际化资源与打包配置。

## 3. 升级步骤
1. 部署后端接口更新。
2. 部署前端公共页与管理端表单改动。
3. 部署 UniApp 新版本。
4. 进入后台维护 App 发行信息、隐私政策和用户政策内容。
5. 在真机上验证协议页、扫码入口与升级检查流程。
6. 在简体中文、繁体中文、英文系统语言下验证 App 名称显示是否正确。
7. 上传前运行 `bash fjbms-uniapp/scripts/check_ios_ipa.sh <ipa-path>` 校验送审包。

## 4. 回滚步骤
1. 回退 FEAT-0031 对应提交。
2. 恢复 UniApp 协议入口到原生内容页。
3. 临时关闭公共路由入口，继续使用现有 App 内页面作为兜底。

## 5. 已知问题
- 本次只按单租户 `d616bcbb` 实现，不适合作为通用公共页方案直接复制到其他租户。
- iOS 下载入口依赖后台配置的 App Store 链接，若未维护则只能展示说明信息。
- iOS 权限文案与图标资源最终以打包产物为准，若 HBuilderX / 云打包配置漂移，需重新核对 `ipa` 解包结果。
- App 名称国际化同样最终以云打包产物为准，若打包链未写入 `locale` 资源，需优先检查新 `ipa` 的本地化应用名称。
