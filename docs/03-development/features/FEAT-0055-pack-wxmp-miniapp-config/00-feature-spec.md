# FEAT-0055 PACK 厂微信小程序配置接入 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-05-28
- related_feature: FEAT-0055
- version: v0.1.0

## 1. 背景与目标
- 客户已确认采用方案一：从 PACK 厂层级接入多个微信小程序。
- 当前 `wx_mp_apps` 按租户唯一，无法支持同一租户下多个 PACK 厂小程序独立接入。
- 当前 APP 内容页已具备按 `apps.appid` 读取能力，适合作为小程序协议和关于我们的内容承载。
- 目标是在不新增 OEM 层级、不调整电池生命周期的前提下，为 PACK 厂配置微信小程序接入信息、首页 Banner、关于我们、隐私协议和用户政策，并让微信小程序端仅保留微信登录注册方式。

## 2. 范围
### In Scope
- 新增 PACK 厂维度微信小程序配置。
- WEB 后台 `BMS 管理 > PACK厂家管理` 增加“小程序配置”入口。
- 支持配置微信 AppID、AppSecret、启停状态、首页 Banner、登录 Logo、备注。
- 复用/创建 `apps` 记录承载内容页，支持 `about_us`、`privacy_policy`、`user_policy` 按小程序单独配置。
- 微信小程序登录按当前小程序 AppID 选择配置并隔离 openid 身份。
- UniApp 微信小程序端隐藏手机号/邮箱登录注册入口，仅保留微信登录。
- 首页 Banner 优先读取 PACK 小程序配置，未配置时使用现有默认图片。
- 登录 Logo 优先读取 PACK 小程序配置，未配置时使用现有默认 Logo。
- APP 管理 > 用户管理支持按 App 端/微信小程序端来源筛选终端用户，并动态展示已配置小程序来源。

### Out of Scope
- 不新增 OEM 客户管理模块。
- 不新增 OEM 类型 WEB 权限、账号、菜单或数据范围。
- 不调整电池出厂、调拨、回退、生命周期层级。
- 不代办微信小程序注册、认证、备案、审核、微信支付配置。
- 不做历史用户跨小程序合并。

## 3. 用户价值
- 租户可给多个 PACK 厂分别配置微信小程序。
- 不同 PACK 厂小程序可展示不同 Banner、关于我们、隐私协议和用户政策。
- 微信小程序端登录入口更符合客户要求，仅使用微信用户体系。
- 保持当前组织和电池业务链路稳定，降低上线风险。

## 4. 验收标准
1. PACK 厂列表可打开“小程序配置”弹窗并保存配置。
2. 非 PACK 组织不能被配置为 PACK 小程序。
3. 同一租户下一个微信 AppID 只能绑定一个 PACK 厂。
4. 小程序配置启用后，微信登录使用对应 AppSecret 调用 code2session。
5. 不同小程序下同一 openid 字符串不会命中同一 `user_identities` 记录。
6. 首页 Banner 按当前小程序配置展示，未配置时保留现有默认图。
7. 登录 Logo 按当前小程序配置展示，未配置时保留现有默认图。
8. 关于我们、隐私协议、用户政策按当前小程序展示。
9. 微信小程序端不显示手机号/邮箱登录、注册、找回密码入口。
10. APP/H5 端原有登录注册入口不受影响。
11. APP 用户管理左侧来源树可动态展示已配置微信小程序，右侧列表按来源筛选并展示身份、设备、访问等细节。

## 5. 风险与约束
- 微信小程序端必须能取得真实微信 AppID；开发者工具、体验版和正式版需分别验证。
- 旧 `wx_mp_apps` 租户级配置需保留兼容，避免影响现有小程序登录。
- `AppSecret` 不应在查询接口返回明文。
- 内容页依赖 `apps` 记录，保存 PACK 小程序配置时需保证关联 App 存在。

## 6. 回滚方案
- 回滚 WEB、UniApp、后端代码。
- 停止读取 `pack_wxmp_configs`，恢复租户级 `wx_mp_apps` 登录配置。
- 保留新增表不读写，避免误删已配置资料；如需彻底回滚，可备份后删除 `pack_wxmp_configs` 及对应测试数据。
