# FEAT-0062 用户质保信息与 PACK 质保卡片开关 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-07-01
- related_feature: FEAT-0062
- version: v0.1.0

## 1. 背景与目标
- 客户二次开发合同已签订，需基于 `docs/business/2026-06-23-warranty-info-quotation.md` 落地移动端用户质保信息、后台电池详情质保 Tab 和电池激活后的质保到期计算。
- PACK 厂需要能在后台配置当前 PACK 小程序是否启用质保卡片；关闭后，该 PACK 小程序移动端质保页仅显示个人质保信息，不显示关联电池卡片。
- 目标是在移动端、后台、后端激活链路之间形成质保信息闭环，并保持既有 PACK 小程序默认开启，避免上线改变现有展示行为。

## 2. 范围
### In Scope
- UniApp “我的 > 质保信息”入口、个人质保联系人维护、电池质保卡片展示。
- 后端用户质保信息保存/读取接口、电池质保查询/编辑接口。
- 电池首次激活时写入质保起始日期，并按 BMS 型号质保月数计算到期日。
- Web 后台电池详情新增“质保”Tab，展示电池质保信息和绑定用户联系人信息，并支持按单块电池编辑。
- PACK 小程序配置增加“启用质保卡片”开关，按当前小程序 AppID 生效。
- SQL 迁移、页面元素权限和 FEAT-0062 文档/看板同步。

### Out of Scope
- 售后工单、维修工单、保养工单流程。
- 按客户、批次、型号的复杂质保规则配置。
- 存量质保数据批量导入或人工清洗。
- 短信、电话外呼、公众号通知。
- 质保统计报表、到期提醒列表或运营看板。

## 3. 用户价值
- 终端用户可在移动端维护个人质保联系人信息，后续售后处理能直接关联到人和联系方式。
- 后台人员可在单块电池详情中查看激活日期、质保时长、到期日和绑定用户信息。
- PACK 厂可控制自己小程序是否展示电池质保卡片，满足不同客户对移动端信息披露范围的要求。

## 4. 验收标准
1. 终端用户登录 UniApp 后，“我的”页展示“质保信息”入口；机构用户不展示该入口。
2. 质保信息页支持姓名、联系电话保存和回显；不强制填写，不阻断其他功能。
3. 当当前 `appid` 对应 PACK 小程序启用质保卡片时，接口返回并展示当前用户关联电池卡片。
4. 当当前 `appid` 对应 PACK 小程序关闭质保卡片时，服务端返回 `batteries: []`，移动端只显示个人质保信息，不显示电池卡片区域和空状态。
5. Web 后台 PACK 小程序配置弹窗支持保存和回显“启用质保卡片”，默认开启。
6. PACK 厂账号只能配置自己组织的 PACK 小程序配置；租户/系统管理员可配置下属 PACK 厂。
7. Web 后台电池详情 BMS 模式新增“质保”Tab，具备页面元素权限控制。
8. 电池首次激活时按 BMS 型号质保月数计算质保到期日；BMS 型号未配置有效月数时不自动生成到期日。
9. 后台人工编辑质保信息后设置 `warranty_manual_override=true`，后续激活流程不覆盖人工到期日。

## 5. 风险与约束
- PACK 质保卡片开关按小程序 AppID 生效，不按电池所属 PACK 逐张过滤。
- 已有 PACK 配置和历史租户默认开启，避免上线后展示行为突然收缩。
- 本地测试无法替代微信小程序真机和后台真实账号权限回归。
- 数据库迁移会扩展 `device_batteries` 和 `pack_wxmp_configs`，发布前需确认 SQL 执行顺序。

## 6. 回滚方案
- 回滚 UniApp 质保页与菜单入口、Web PACK 配置开关和电池详情质保 Tab。
- 回滚后端新增质保接口、激活链路质保计算和 PACK 配置字段使用。
- 数据库层如需回滚，可保留新增列和表不使用；若必须清理，需在确认无线上数据依赖后删除 `user_warranty_infos` 与新增列。
