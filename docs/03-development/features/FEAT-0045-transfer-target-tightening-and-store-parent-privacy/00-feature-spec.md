# FEAT-0045 调拨目标收敛与门店上级组织防泄露 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-21
- related_feature: FEAT-0045
- version: v0.1.0

## 1. 背景与目标
- 电池列表调拨弹窗当前对经销商仍展示 `PACK厂/门店`，与“只下发到下一级”的口径不一致。
- 门店账号仍可发起调拨请求，存在“界面可点、业务不应允许”的越权风险。
- 门店管理新增弹窗当前会拉取完整组织树，导致经销商场景可见全量组织结构。

本次目标：
1. 经销商调拨只允许目标门店；门店账号不允许调拨。
2. 经销商新增门店时，上级组织只读为当前机构，不请求组织树。
3. 后端同步收口，防止绕过前端发请求。

## 2. 范围
### In Scope
- `BMS 管理 > 电池列表` 调拨目标类型收敛与门店入口禁用提示。
- `POST /api/v1/battery/transfer` 权限矩阵收紧：
  - 经销商仅允许调拨到下级门店；
  - 门店无调拨权限。
- `BMS 管理 > 门店管理` 新增门店时上级组织改为只读当前机构。
- `POST /api/v1/org` 增加经销商创建门店的 `parent_id` 强校验。
- 文档与看板同步更新。

### Out of Scope
- 不改变 BMS 厂家/PACK 厂家的调拨路径。
- 不新增新的组织树查询接口。
- 不改造批量调拨与审批流。

## 3. 验收标准
1. 经销商登录电池列表调拨弹窗，仅显示“门店”目标类型。
2. 门店登录时“调拨”入口保留但不可执行，点击后仅提示无可调拨下级机构。
3. 门店账号调用 `/battery/transfer` 被后端拒绝并返回明确错误。
4. 经销商调用 `/battery/transfer` 传入非门店目标被后端拒绝。
5. 经销商进入“门店管理 > 新增门店”时，上级组织为只读当前机构，页面不请求 `getOrgTree`。
6. 经销商调用 `/org` 创建门店时，`parent_id != claims.OrgID` 被后端拒绝。

## 4. 风险与约束
- 前端入口禁用只改善体验，最终以后端权限校验为准。
- 组织树防泄露通过“经销商新增门店场景不调用树接口”实现，其他角色保持现状。
- FEAT-0028 中旧调拨矩阵需标注为历史口径，避免团队继续按旧规则开发。

## 5. 回滚方案
- 回滚前端电池调拨选项收敛和门店管理上级组织只读逻辑。
- 回滚后端 `TransferBattery` 与 `CreateOrg` 的新校验分支。
- 保留历史转移记录，不做数据回退。
