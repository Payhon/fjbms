# FEAT-0006 UniApp 升级检测与提示体验优化 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-10
- related_feature: FEAT-0006
- version: v0.1.0

## 1. 背景与目标
- 当前 `fjbms-uniapp` 设备详情页仅在用户进入“参数设置”并点击 OTA 版本项时才发起升级检测，提示时机过晚。
- 需要将升级检测前置到设备详情页默认展示的“仪表盘”阶段，让用户在未进入参数页时也能感知有新固件可升级。
- 当检测到新版本时，设备详情页底部“参数设置”一级 Tab 与参数页 OTA 版本项都要显示红点提示。
- 用户进入参数设置页后，若存在新版本，OTA 固件升级项需在红点旁以灰色字体显示“新版本：{版本号}”。

## 2. 范围（In Scope / Out of Scope）
- In Scope
  - 设备详情页进入仪表盘后自动触发一次非阻塞 OTA 检测。
  - OTA 检测结果在设备详情页内共享，用于驱动底部 Tab 与参数页 OTA 行红点。
  - 保留手动点击 OTA 行的升级入口，并兼容未完成自动检测时的兜底查询。
  - OTA 成功后清除红点；后续复检确认无需升级时也清除红点。
- Out of Scope
  - 不修改后端 OTA 检测接口协议。
  - 不新增全局缓存或跨页面持久化红点状态。
  - 不调整全局 `custom-tab-bar`，仅修改设备详情页底部 Tab。

## 3. 用户价值
- 用户进入设备详情页即可看到有无可升级版本，减少遗漏升级的概率。
- 红点提示位置与操作入口一致，用户可以快速定位到参数设置中的 OTA 项。
- 保持仪表临时会话下隐藏 OTA 的既有约束，避免无 `device_id` 场景误触发云端检查。

## 4. 验收标准
1. 云端设备进入设备详情页且默认停留在仪表盘时，自动触发一次 OTA 检测，无需点击版本项。
2. 当 OTA 检测返回 `need_upgrade=false` 时，底部“参数设置”Tab 与参数页 OTA 行都不显示红点。
3. 当 OTA 检测返回 `need_upgrade=true` 时，底部“参数设置”Tab 与参数页 OTA 行都显示红点。
4. 用户切换 Tab 后返回，红点状态保持一致，不因进入参数页或点击 OTA 行而自动消失。
5. OTA 成功后清除两处红点，并允许后续基于新版本再次复检。
6. 仪表临时会话模式不触发自动 OTA 检测，也不显示 OTA 红点。
7. OTA 自动检测失败时页面不弹错误 toast，不阻塞仪表盘展示；手动点击 OTA 行仍可重试检查。

## 5. 风险与约束
- 设备详情页的版本号与型号信息来自异步加载数据，自动检测需要等待必要字段就绪后再发起。
- 若 OTA 成功后设备版本上报存在延迟，页面内立即复检可能仍看到旧版本；本次以“清除当前红点并允许下一轮检查”作为收敛策略。
- 同一页面生命周期内需要对相同 `(device_id, model, version)` 做去重，避免重复请求。

## 6. 回滚方案
- 回滚 `fjbms-uniapp/pages/device-battery/detail.vue` 中页面级 OTA 检测与底部 Tab 红点逻辑。
- 回滚 `fjbms-uniapp/pages/device-battery/components/params-tab.vue` 中 OTA 共享状态消费与红点展示逻辑。
- 保持现有“点击 OTA 版本项时再检测”的原始行为。
