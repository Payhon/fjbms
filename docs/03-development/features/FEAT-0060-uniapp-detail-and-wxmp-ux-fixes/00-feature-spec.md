# FEAT-0060 UniApp 详情状态与小程序品牌首屏优化 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-06-18
- related_feature: FEAT-0060
- version: v0.1.0

## 1. 背景与目标
- 4G BMS 设备详情页顶部连接状态只按 `connType=mqtt` 显示“已连接”，当首页设备列表已显示离线时，详情页仍显示“4G 已连接”，容易让用户误解设备在线。
- 首页设备列表 4G 设备状态角标仍显示“MQTT 在线”，对用户暴露了技术链路名称，也无法明确区分 4G 在线/离线。
- Android APP 从底部中间添加入口选择扫码/相册识别后，若从相册二维码跳转到设备详情，添加设备底部菜单可能因中间 Tab 页面重入再次出现。
- PACK 厂微信小程序冷启动时，运行配置接口返回前会短暂显示富嘉默认 Logo/Banner，存在品牌闪现问题。
- 目标是在不调整后端接口、协议和在线判定规则的前提下，修正移动端状态展示、扫码菜单生命周期和 PACK 小程序首屏品牌展示。

## 2. 范围
### In Scope
- UniApp 设备详情顶部连接状态文案、样式和中英文翻译。
- UniApp 首页设备卡片 4G 状态文案、样式和中英文翻译。
- UniApp APP 端添加设备 ActionSheet 的扫码流程保护和中间 Tab 回退。
- UniApp 微信小程序首页 Banner、登录 Logo 的运行配置加载前后展示策略。
- FEAT-0060 文档与项目看板同步。

### Out of Scope
- 不新增或修改后端接口。
- 不调整数据库结构、设备在线判定规则或 MQTT/BLE 协议。
- 不改动 WEB 后台 PACK 小程序配置表单。

## 3. 用户价值
- 4G 设备在线/离线状态在首页和详情页语义一致，减少误操作。
- Android APP 扫码后进入设备详情不会残留添加设备菜单。
- PACK 厂小程序冷启动不暴露富嘉默认品牌资产。

## 4. 验收标准
1. 4G 在线设备详情顶部显示“4G 在线”并使用绿色状态。
2. 4G 离线设备详情顶部显示“4G 离线”并使用醒目离线色。
3. 首页设备列表 4G 在线设备显示“4G 在线”，4G 离线设备显示“4G 离线”，不再显示“MQTT 在线”。
4. BLE 设备仍按原逻辑显示蓝牙连接/离线。
5. Android APP 从底部中间添加入口选择扫码/相册识别并从相册二维码跳转详情后，底部添加菜单不残留、不重弹。
6. PACK 小程序运行配置返回前不显示具体 Logo/Banner。
7. PACK 小程序未配置专属 Logo/Banner 时继续隐藏，配置后展示专属图。
8. 非 PACK 小程序配置返回且未配置专属图时可回退默认图。

## 5. 风险与约束
- 4G 在线/离线展示只消费现有 `is_online` 字段，不改变后端在线口径。
- 微信小程序无法在本地 TypeScript 校验中完整模拟 AppID 运行配置，需微信开发者工具或真机回归。
- Android 相册识别回跳依赖宿主生命周期，需真机确认 ActionSheet 不再重入。

## 6. 回滚方案
- 回滚 `fjbms-uniapp` 中 FEAT-0060 涉及的设备详情、添加设备入口、首页和登录页改动。
- 若 PACK 首屏隐藏策略影响非 PACK 小程序，可优先回退首页/登录页初始默认图策略，保留 4G 状态和扫码菜单修复。
