## 2. 关键决策
- 使用 `authority + user_kind + is_main` 定义后台用户主从体系。
- 角色业务表作为权限事实来源，Casbin 继续承担运行时授权适配。
- 登录后权限遵循“主账号基础全集、子账号基础范围内角色并集”的规则。

## 3. 风险跟踪
- 历史数据回填可能误判主账号，需要以“最早创建记录”作为默认规则并保留复核空间。
- 多处租户识别逻辑需统一改造，避免出现口径不一致。
- 前端完整 `pnpm build` 在当前环境未拿到最终结束结果，已先通过后端编译与前端定向 ESLint 校验收敛显式错误。
