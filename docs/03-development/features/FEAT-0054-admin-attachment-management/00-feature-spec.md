# FEAT-0054 后台附件管理 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-05-19
- related_feature: FEAT-0054
- version: v0.1.0

## 1. 背景与目标
- 后台上传组件生成的附件已记录到 `files` 表，但没有统一的后台清理入口，长期会造成本地或云存储对象无法管理。
- 目标是在后台管理“系统管理”下新增附件管理，支持按当前租户查看、筛选、预览、下载和删除附件。

## 2. 范围
### In Scope
- 后台新增附件管理菜单与页面。
- 按当前租户列出全部附件，不限制为当前用户上传。
- 支持关键词、业务类型、存储位置、上传时间范围筛选。
- 支持预览、下载、删除附件。
- 删除时同步删除本地文件或云存储对象。
- 新增菜单权限种子，纳入菜单管理和角色权限配置。

### Out of Scope
- 不新增附件主表。
- 不做 SYS_ADMIN 跨租户附件总览。
- 不扫描业务字段引用关系。
- 不迁移历史 `files` 数据。

## 3. 验收标准
1. 系统管理下可看到“附件管理”菜单，且菜单受现有权限体系控制。
2. 附件管理可分页查看当前租户全部附件。
3. 筛选条件可按关键词、业务类型、存储位置和上传时间范围生效。
4. 图片附件可弹窗预览，非图片附件可新窗口打开。
5. 本地附件可通过管理接口下载，云存储附件可打开现有访问 URL。
6. 删除附件前显示强确认，确认后同步删除存储对象并移除 `files` 记录。
7. 云存储删除失败时保留数据库记录，并向前端返回失败。
8. 现有上传组件和 `FilePicker` 的“我的文件”行为不变。

## 4. 风险与约束
- 删除不检测业务引用，可能导致已引用附件的页面或记录无法继续访问。
- 云存储删除依赖当前系统存储配置；配置错误时删除会失败并保留数据库记录。
- 本地文件删除仅允许 `./files/` 下路径，避免误删服务目录外文件。

## 5. 回滚方案
- 回滚前后端代码。
- 回滚增量 SQL 中新增的 `management_attachment` 菜单记录。
- 已删除的存储对象无法自动恢复，需从备份或云存储回收策略恢复。
