# FEAT-0036 后台短信验证码调试能力 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-04-07
- related_feature: FEAT-0036
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - 当前移动端在注册页点击“获取验证码”时，前端只收到统一错误文案“请求参数验证失败”。
  - 后端 `app/auth/phone/code` 链路内部会把“模板未配置”“短信服务未开启”“模板ID为空”“短信供应商返回失败”等多类问题都折叠为 `CodeParamError` 或通用错误码，页面无法直接识别真实失败环节。
  - 后台通知配置页目前仅支持邮件调试，不支持短信验证码链路排查。
- 目标：
  1. 在后台 `系统管理 > 通知设置 > 短信` 页面增加短信调试入口。
  2. 对 `SYS_ADMIN` 增加租户下拉选择，确保调试时可显式指定目标租户。
  2. 调试时走与移动端手机号注册验证码一致的完整链路，包括：
     - 租户短信模板读取
     - 验证码生成与缓存
     - 通知配置短信通道发送
  3. 将每一步诊断结果展示给管理员，便于快速定位失败环节。

## 2. 范围
### In Scope
- `frontend/src/views/management/notification/components/short-message.vue`
- `frontend/src/service/api/notification-services.ts`
- `backend/internal/api/notification_services_config.go`
- `backend/internal/service/app_auth.go`
- `backend/internal/service/notification_services_config.go`
- 调试接口的数据模型、返回结构、文档与看板回写。

### Out of Scope
- 不修改移动端注册页交互。
- 不改变正式注册发码接口的返回码规范。
- 不新增新的短信供应商。

## 3. 用户价值
- 管理员可直接在后台输入手机号并测试注册验证码真实链路。
- 当短信验证码发送失败时，可以明确知道是：
  - 租户短信模板未配置或未开启
  - 通知配置短信服务未开启
  - 短信模板 ID 缺失或错误
  - 阿里云短信发送失败
- 减少靠日志和数据库排查的时间成本。

## 4. 验收标准
1. 短信配置页新增“调试”按钮，点击后可输入手机号和验证码场景。
2. `SYS_ADMIN` 打开调试弹窗时必须可以先选择租户。
3. 选择 `REGISTER` 场景并发送后，后台调用与移动端注册验证码一致的完整链路。
4. 页面结果区需展示每个调试步骤的成功/失败和详细说明。
5. 当短信发送成功时，结果区需包含供应商、签名、模板 ID、RequestId 等关键诊断信息。
6. 本次改动不引入前端 TypeScript 或后端编译错误。

## 5. 风险与约束
- 调试接口会触发真实短信发送，需要由管理员明确输入接收手机号。
- APP 认证短信模板为租户维度配置，若当前管理员上下文租户与移动端注册租户不一致，调试结果不能代表目标租户链路。
- 不应在调试结果中暴露短信密钥等敏感字段。

## 6. 回滚方案
- 回滚短信调试接口、短信页调试弹窗及 FEAT-0036 文档与看板记录。
