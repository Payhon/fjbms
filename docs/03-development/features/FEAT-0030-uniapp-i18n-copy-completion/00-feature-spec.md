# FEAT-0030 UniApp 页面多语言文案补全 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-02
- related_feature: FEAT-0030
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - `fjbms-uniapp` 已接入 `vue-i18n`，但首页、设备详情及“我的”等页面仍存在用户可见文案硬编码、部分英文缺失、以及混用中文常量的情况。
  - 当前问题会导致语言切换后页面仍残留中文，且新旧页面文案口径不统一。
- 目标：
  1. 从首页开始，补齐首页、设备详情及其他常用页面的多语言调用。
  2. 将页面内用户可见硬编码文案统一替换为 `vue-i18n` key。
  3. 同步补充 `zh-CN` / `en-US` 字典缺失项，保证切换语言后页面展示完整。
  4. 在不改变业务逻辑与交互行为的前提下，清理通用模块中的提示文案硬编码。

## 2. 范围
### In Scope
- `fjbms-uniapp/pages/home/` 首页及其相关组件的用户可见文案补齐。
- `fjbms-uniapp/pages/device-battery/` 设备详情页及子组件的用户可见文案补齐。
- `fjbms-uniapp/pages/alarm/`、`pages/my/`、`pages/notify/`、`pages/org-devices/` 等常用页面的用户可见文案补齐。
- `fjbms-uniapp/common/request.ts`、`common/util.ts` 等通用提示模块的用户可见 toast / modal 文案补齐。
- `fjbms-uniapp/lang/zh-CN.ts`、`fjbms-uniapp/lang/en-US.ts` 新增缺失字典项。
- 功能文档、看板、测试记录同步回写。

### Out of Scope
- 不修改后端接口返回字段与业务协议。
- 不改动 `uni_modules/` 第三方组件包内部文案。
- 不进行页面视觉样式调整或交互重构。
- 不处理纯注释、开发日志、调试输出中的中文内容。

## 3. 用户价值
- 语言切换后首页、设备详情和主要业务页不再残留硬编码中文。
- 中英文口径统一，减少用户理解偏差和后续维护成本。
- 为后续继续补齐其他冷门页面提供统一命名空间与实践基线。

## 4. 验收标准
1. 首页可见文案均通过 `$t(...)` / `t(...)` / `i18n.t(...)` 输出，不再直接展示硬编码中文。
2. 设备详情页及仪表盘、参数、电芯、历史记录子页中的可见文案均支持中英文切换。
3. “我的”、告警、通知详情、机构设备等常用页面的关键用户可见文案支持中英文切换。
4. 通用 toast / modal 等运行时提示不再残留硬编码中文。
5. `zh-CN.ts` 与 `en-US.ts` 新增的 key 能覆盖本次改造页面，不出现明显缺词或 key 原样透出。

## 5. 风险与约束
- 历史页面存在 Options API、Composition API 与旧 uni-app 写法混用，改造时必须保持原有行为不变。
- 多个页面复用相近文案，需要统一 key 命名，避免同义重复。
- 设备详情页涉及协议状态映射，需区分“展示文案”与“协议值”边界，避免误改业务判断。

## 6. 回滚方案
- 回滚 `fjbms-uniapp` 本次涉及页面和组件的 i18n 调用改动。
- 回滚 `zh-CN.ts` / `en-US.ts` 中新增字典项。
- 回滚看板与本功能文档状态。
