# FEAT-0046 仪表 OTA 升级包与 UniApp 独立升级链路 - 发布说明

- status: in_progress
- owner: payhon
- last_updated: 2026-04-28
- related_feature: FEAT-0046
- version: v0.1.0

## 1. 本次变更
- OTA 升级包管理新增 `BMS 升级包 / 仪表升级包` 双 TAB。
- 新增仪表升级包类型，后台可单独上传和维护仪表固件。
- UniApp 蓝牙仪表详情页新增“仪表升级”卡片，支持手动选择仪表固件并直接升级。
- 修复 Android 仪表扫描进入详情可能连接到历史 AC 设备的问题。
- 修复 Android BLE 连接 API 不回调时详情页长期停留在“连接中”的问题。
- 优化 Android 仪表 OTA 写包节奏和超时，增加 4KB 固件边界等待，避免边界擦写期间提前判定失败；保持 Boot 帧单次 BLE 写入，兼容仪表蓝牙透传模块。
- 优化 Android 仪表 OTA 完成阶段：全部数据包 ACK 后，`0x54` finalize 导致设备重启/断开时按成功收敛，避免误报超时告警。
- 优化蓝牙扫描页停止与超时体验：停止后不再处理过期设备发现回调，扫描超时后自动停止动画并恢复“开始扫描”按钮。
- 修复 iOS 仪表 OTA 末段 0x53 超时后重试无法继续的问题。
- 修复 iOS 仪表 OTA 全部数据包写完后，`0x54` finalize 触发设备断开却被误判为升级失败的问题。
- 修复微信小程序仪表 OTA 误报成功：仪表升级必须完成全部 `0x53` 数据包 ACK，等待 2s 后发送 `0x54`，且收到 `0x54 status=0` 后才显示升级完成。
- 提升蓝牙 BMS 与蓝牙仪表 OTA 完成阶段稳定性：对齐旧版小程序经验，`0x54` 完成指令会在短时间内补发多次；BMS 与仪表均必须收到 `0x54 status=0` 才判定成功，仪表额外兼容 `0xFD` Boot 回包源地址。
- 蓝牙 BMS 与蓝牙仪表均按协议严格闭环完成升级；Android 仪表的慢速写包参数仍只作用于仪表升级。
- 优化 Android 仪表 OTA 写包速度：`0x53` 默认改为 ACK 驱动低延迟传输，减少固定包间等待和 4KB 边界等待；如遇 `0x53` 超时会自动降速保护。
- 优化 iOS 从蓝牙扫描页进入仪表详情的首连耗时：扫描页 UUID 型 `deviceId` 可直接作为连接候选，主动取消旧连接后不再长时间等待旧连接锁。
- 开发者模式下新增仪表 OTA 端上调试日志，可在参数页直接查看并复制，便于排查用户环境下连接、下载、`0x50/0x52` 前置失败。
- 优化微信小程序主包体积：仪表 OTA 调试日志模块移入设备详情分包，并移除未引用的静态 `moment.js`。
- `0x53` 回包 `status=5` 统一提示为固件包大小与设备不一致。

## 2. 影响范围
- `backend/` OTA 升级包模型与 APP 仪表包接口
- `frontend/` 后台 OTA 升级包管理页
- `fjbms-uniapp/` 设备详情参数页 OTA 入口

## 3. 升级步骤
1. 执行 `backend/sql/51.sql`。
2. 发布后端服务。
3. 发布后台前端。
4. 重新打包并发布 UniApp 客户端。

## 4. 回滚说明
- 回滚 `device_kind` 接口逻辑与后台双 TAB 页面。
- UniApp 回滚为原有 BMS OTA 入口。
- 如需保留数据，可仅停止前端入口并保留仪表包记录。
