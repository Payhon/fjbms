# FEAT-0018 UniApp 温度参数支持负数输入 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-03-20
- related_feature: FEAT-0018
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - 移动端电池详情页“参数设置”中的编辑弹窗，当前非字符串参数统一使用数字键盘。
  - 温度相关参数存在负数场景，但当前键盘无法输入 `-`，导致低温阈值类参数无法正确设置。
- 目标：
  1. 温度类参数编辑时允许输入负数。
  2. 非温度数值参数继续保持原有数字键盘行为，减少对其它参数设置流程的影响。

## 2. 范围
### In Scope
- 调整 `fjbms-uniapp/pages/device-battery/components/params-tab.vue` 中参数编辑弹窗的输入类型选择逻辑。
- 基于参数单位/键名识别温度类输入项，并切换到允许输入负号的输入模式。

### Out of Scope
- 不改动后端参数写入协议。
- 不改动 Web 端参数编辑行为。

## 3. 验收标准
1. 温度单位为 `℃` 的参数项打开编辑弹窗后，可以输入 `-10`、`-5.5` 等负数。
2. 非温度数值参数仍维持原有数字键盘。
3. 弹窗确认时仍沿用现有数值校验逻辑，非法输入不会提交。

## 4. 风险与约束
- 纯文本键盘虽然允许输入负号，但也会放开其它字符输入，需要依赖现有确认时数值校验兜底。
- 若后续存在非温度但也允许负数的参数，需要扩展当前识别规则。

## 5. 回滚方案
- 回滚 `params-tab.vue` 的输入类型切换逻辑，恢复所有非字符串参数统一使用数字键盘。
