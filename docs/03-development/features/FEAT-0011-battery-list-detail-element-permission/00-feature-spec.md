# FEAT-0011 电池列表详情路由与元素权限补齐 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-03-08
- related_feature: FEAT-0011
- version: v0.1.0

## 1. 背景与目标
- 背景：机构用户（PACK 厂家/经销商/门店）已具备“电池列表”菜单权限，但点击“操作 -> 查看详情”会因缺少详情页路由权限而无法进入。
- 目标：
  1) 补齐电池列表下属详情页路由权限。
  2) 补齐电池列表页面关键操作按钮/菜单权限（导出、添加电池、导入、参数、离线指令、生命周期：出厂/激活/调拨）。
  3) 通过 Vue 指令实现页面元素权限控制，默认无权限隐藏，支持按属性配置为“显示但禁用”。

## 2. 范围
### In Scope
- 后端：新增当前用户 UI 权限查询接口；菜单权限配置树支持按钮节点。
- SQL：新增 `sys_ui_elements` 路由/按钮权限项；为已授予电池列表权限的机构类型补齐新增权限编码；同步 `1.sql`。
- 前端：新增通用权限指令并在电池列表页面接入按钮/操作项权限控制。

### Out of Scope
- 不改动现有角色模型与 Casbin 策略结构。
- 不改动电池详情页业务逻辑。

## 3. 验收标准
1. 机构用户具备 `bms_battery_list_detail` 权限时，可从电池列表进入设备详情页。
2. 电池列表页面中以下元素按权限控制：
   - 顶部按钮：导出、添加电池、导入。
   - 操作菜单：参数、离线指令、生命周期（出厂/激活/调拨）。
3. 无权限默认隐藏；指令启用“禁用模式”时无权限元素可见但不可点击。

## 4. 风险与约束
- 历史租户 `org_type_permissions.ui_codes` 可能未及时补齐新增编码，需执行 SQL 补丁。
- 若权限配置页面不包含按钮节点，后续手工保存配置可能覆盖按钮权限，需确保后端返回可选按钮节点。

## 5. 回滚方案
- 回滚前端权限指令接入与页面改动。
- 回滚后端新接口与按钮节点暴露逻辑。
- 回滚 SQL 补丁 `40.sql`（删除新增 `element_code` 并恢复租户 `ui_codes` 备份）。
