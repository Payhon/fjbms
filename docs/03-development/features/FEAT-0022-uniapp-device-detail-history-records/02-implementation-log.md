# FEAT-0022 UniApp 设备详情页历史记录功能 - 实施日志

- status: review
- owner: payhon
- last_updated: 2026-03-22
- related_feature: FEAT-0022
- version: v0.1.0

## 日志
1. 2026-03-22 协议分析
   - 读取并确认 `doc/oriigin/历史记录读取协议.md` 中 `0x4C / 0x4D` 指令、字段含义、单次读取上限与状态位定义。
   - 确认当前 UniApp 协议库尚未提供历史记录接口，但 `BmsClient` 具备可复用的底层请求能力。
2. 2026-03-22 协议库扩展
   - 在 `common/lib/bms-protocol/frame.ts` 中新增 `HISTORY_PROTECTION_COUNT`、`HISTORY_STATUS_RECORD`。
   - 新增 `history-parser.ts`，实现历史保护次数与历史状态记录的载荷解析。
   - 在 `types.ts`、`client.ts`、`index.ts` 中新增公开类型与读取方法。
3. 2026-03-22 设备详情页改造
   - `detail.vue` 已新增第 4 个底部一级菜单“历史记录”，并将页面滚动触底转发给历史记录组件分页加载。
   - 参数设置与历史记录统一接入暂停轮询逻辑。
4. 2026-03-22 历史记录组件
   - 新增 `pages/device-battery/components/history-tab.vue`。
   - 实现保护次数摘要卡、分组计数列表、状态记录卡片、状态标签、空态/异常态与加载态。
   - 状态记录按最新优先展示，首屏 6 条，支持继续向前加载。
5. 2026-03-22 文案与文档
   - 在 `lang/zh-CN.ts`、`lang/en-US.ts` 中补充 `deviceDetail.history` 全量文案与映射。
   - 新增 FEAT-0022 文档集并更新项目看板。
6. 2026-03-22 联调修正
   - 修复历史保护次数列表中 `{n}` 未被替换为实际次数的问题，改为兼容 message compiler 被裁剪场景的手动占位替换。
   - 在历史记录读取链路中增加控制台日志，覆盖 `0x4C / 0x4D` 的发送帧、返回帧、解析结果、异常与回退路径。
   - 对 `0x4D` 状态记录读取增加“批量失败后按单条顺序回退”的兜底逻辑，便于兼容对多条读取超时的设备。
7. 2026-03-22 再次联调修正
   - 修复状态记录卡片中的 `第{n}条`、`事件{code}` 占位符未替换问题。
   - 按联调反馈将状态记录分页策略调整为“每次只读取 1 条记录”，降低 BLE 大包超时概率。
8. 2026-03-22 历史记录界面继续优化
   - 根据协议语义将二级页签文案调整为“总计 / 明细”，其中 `0x4C` 对应总计，`0x4D` 对应逐条明细。
   - 状态记录卡片补充显示 `17-20` 字节的剩余容量，完成 32 字节明细包全部有效字段的前端呈现；`23-26` 字节状态位继续通过标签区展示。
   - 为 `日志1` 事件码补充设备联调中出现的 `4 -> 上电启动` 映射，并同步英文翻译。
   - 明细区域改为组件内 `scroll-view` 滚动容器，同时保留页脚点击加载更多兜底，解决首屏仅 1 条时页面整体不触发 `onReachBottom` 的问题。
9. 2026-03-22 按完整版协议回补
   - 重新对照 `doc/oriigin/历史记录读取协议(完整版).md`，补齐 `日志1` 中此前缺失的多段枚举值，并修正 `89` 的文案为“0 电流校准”。
   - 按完整版协议说明移除地址 `26` 中 `BIT4/BIT5` 的前端解析与展示，仅保留“正在放电 / 正在充电”两项状态位。
   - 同步更新中英文词典、事件徽标色彩分组和技术设计文档，确保协议定义、解析实现与 UI 展示一致。
10. 2026-03-22 按源截图校正状态位
   - 根据用户提供的源 PDF 截图，校正地址 `23` 的电压状态展示文案：`BIT0/BIT1` 改为“过放恢复”，`BIT7` 改为“电压过压保护”。
   - 在 `doc/oriigin/历史记录读取协议(完整版).md` 中补充说明：源图中 `BIT1` 与 `BIT3` 均标注为“电芯过放恢复”，文档按原图保留。
11. 2026-03-22 明细头部交互优化
   - 将“总条数 / 已加载 / 当前地址”合并为明细滚动区内的粘性头部，滚动历史卡片时持续固定在顶部。
   - 新增当前地址展示，并压缩为三列单行布局，减少明细页重复信息的视觉跳动。
12. 2026-03-25 明细排序修正
   - 修正 `0x4D` 明细分页方向，从索引 `0` 开始顺序读取，并在前端按记录时间倒序展示。
   - 明细列表现改为“最近时间优先”，上滑加载时继续追加更早的历史记录。
13. 2026-03-25 明细分页修正
   - 修复首屏替换加载时 `resetStatusState()` 误清空 `totalCount`，导致 `hasMore` 被错误计算为 `false` 的问题。
   - 保持“每次读取 1 条”的前提下，新增首屏自动补载逻辑，至少补到 2 条，避免内容不足一屏时无法通过上滑触发继续加载。
14. 2026-03-25 明细顺序修正
   - 移除明细列表按时间字段做的二次全量排序，改为严格按历史索引顺序显示，避免分页追加后卡片序号与列表顺序错乱。
   - 合并分页结果时按索引去重，确保首屏补载和后续滚动加载不会插入重复记录。
15. 2026-03-25 明细地址动态化
   - 将明细头部中的“当前地址”从固定显示 `0x4C` 汇总地址改为“当前已加载到的明细地址”。
   - 地址值基于 `0x4C.currentRecordAddress` 与当前列表最后一条已加载记录索引推导，随着继续上滑加载更早记录而动态变化。
