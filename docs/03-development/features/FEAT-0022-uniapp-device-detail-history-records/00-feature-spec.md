# FEAT-0022 UniApp 设备详情页历史记录功能 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-03-29
- related_feature: FEAT-0022
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - 当前 `fjbms-uniapp` 设备详情页仅有“仪表盘 / 电芯 / 参数设置”三类信息，缺少设备端历史记录入口。
  - 协议文档 `doc/oriigin/历史记录读取协议.md` 已定义 `0x4C` 历史保护次数和 `0x4D` 历史状态记录，但移动端尚未接入。
- 目标：
  1. 在设备详情页底部新增一级菜单“历史记录”。
  2. 在历史记录页内提供“保护次数记录 / 状态记录”二级切换。
  3. 以移动端高可读、低操作成本的方式展示历史记录，并兼容蓝牙、MQTT、Socket Bridge 与仪表临时连接模式。
  4. 历史记录入口是否显示，需根据当前用户类型对应的移动端权限配置决定。

## 2. 范围
### In Scope
- 扩展 UniApp 协议库以支持 `0x4C / 0x4D` 历史记录读取与解析。
- 新增历史记录组件，并按权限接入设备详情页底部一级 Tab。
- 实现保护次数摘要卡、分组列表、状态记录时间卡片、状态标签、加载/空/错误态。
- 补充中英文多语言文案、日志码映射、状态位标签映射。
- 后台“系统管理 > 权限管理”新增“移动端权限”配置页签，并提供“历史记录”权限项。
- 后端新增移动端专用权限接口，按当前用户类型独立返回历史记录权限。
- 新增 FEAT-0022 文档并更新项目看板。

### Out of Scope
- 不改后台历史记录查询逻辑。
- 不新增导出、筛选、搜索或日期范围过滤。
- 不修改现有设备详情页的仪表盘、电芯、参数设置业务逻辑。

## 3. 验收标准
1. 已分配“历史记录”移动端权限的用户进入设备详情页时，底部显示“历史记录”一级菜单并可正常切换。
2. 未分配“历史记录”移动端权限的用户进入设备详情页时，不显示“历史记录”一级菜单。
3. 历史记录页内支持“保护次数记录 / 状态记录”二级切换，激活态明显。
4. 切入“参数设置”或“历史记录”时暂停状态轮询，离开后恢复。
5. 保护次数记录可展示总记录数、当前记录地址、累计充/放电时长及分组计数项。
6. 状态记录默认按最新优先显示，首屏最多读取 6 条，滚动到页面底部可继续加载更早记录。
7. 无连接、无记录、不支持、读取失败四类状态均有明确反馈。
8. 仪表临时连接模式下，若当前账号有权限，则可进入历史记录页签；若无权限则直接隐藏入口。

## 4. 风险与约束
- 文档提供的 `0x4C` 样例载荷长度与 CRC 口径存在不一致，解析实现需优先保证字段容错，不因样例尾部预留区差异而失败。
- 历史记录读取与状态轮询共用同一设备连接，必须串行请求，避免蓝牙链路超时或回复串包。
- 该页面需兼容 iOS、Android、微信小程序三端，不能依赖平台特有 UI 能力。
- 当前移动端权限需严格按用户类型独立判断，机构用户不能复用 `APP_USER` 的“历史记录”权限。

## 5. 回滚方案
- 回滚 `fjbms-uniapp/pages/device-battery/detail.vue` 中历史记录权限判断、一级菜单与组件接线。
- 回滚 `fjbms-uniapp/common/lib/bms-protocol` 中 `0x4C/0x4D` 协议扩展。
- 回滚后台权限管理页新增的“移动端权限”页签与后端 `/api/v1/org_type_permissions/mobile_ui_codes/me` 接口。
- 回滚 `lang/zh-CN.ts`、`lang/en-US.ts` 中新增的 `deviceDetail.history` 文案。
