# FEAT-0042 电池出厂增强与跨页批量选择 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-17
- related_feature: FEAT-0042
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - 第三方 MES 电池建档接口当前只能新增/更新电池档案，无法按 PACK 厂家名称自动完成出厂。
  - 后台电池列表已有单条“出厂”操作，但缺少批量出厂入口，且表格勾选仅在当前分页有效。
- 目标：
  1) `POST /api/v1/openapi/mes/battery` 支持传入 `pack_factory_name`，自动解析 PACK 厂家并出厂。
  2) 后台电池列表支持批量出厂。
  3) 电池列表批量选择支持跨分页保留，并覆盖现有批量操作。

## 2. 范围
### In Scope
- OpenAPI 新增 `pack_factory_name` 可选字段。
- 后端新增批量出厂接口，返回成功/失败汇总。
- 前端批量操作新增“批量出厂”入口。
- 电池列表勾选状态改造为跨分页保留。
- 同步权限、多语言、第三方接入文档、开发文档与项目看板。

### Out of Scope
- 不改动 UniApp 页面与移动端交互。
- 不新增独立“批量出厂记录”页面，继续复用现有运营日志/转移记录。
- 不支持通过 PACK 厂家 ID 直接传值，仍以名称匹配。

## 3. 验收标准
1. OpenAPI 请求体新增 `pack_factory_name`，名称可为空。
2. 当 `pack_factory_name` 唯一匹配当前租户 `orgs.name` 且 `org_type=PACK_FACTORY` 时，电池建档完成后自动出厂到该 PACK 厂家。
3. 当 `pack_factory_name` 未匹配或匹配不唯一时，电池建档仍成功，但自动出厂跳过，并保留可追踪日志。
4. 后台电池列表批量操作菜单新增“批量出厂”。
5. 批量出厂支持选择目标类型 `PACK_FACTORY/DEALER` 与目标机构，并返回逐台成功/失败汇总。
6. 跨页勾选后，批量操作数量、摘要信息和提交对象一致，不因翻页丢失。
7. 更换筛选条件或重置查询后，历史勾选会清空，避免跨条件误操作。
8. 新权限码 `bms_battery_list_batch_factory_out` 生效，并在前端菜单显隐一致。

## 4. 风险与约束
- `pack_factory_name` 基于租户内精确名称匹配；若存在同名 PACK 厂家，会导致自动出厂跳过。
- 批量出厂采用部分成功模式，前端必须展示失败明细，避免用户误认为全部成功。
- 跨页选择缓存依赖当前已加载行数据补齐展示字段；批量摘要以已选缓存为准，不额外查询后端详情。

## 5. 回滚方案
- 回滚前端批量出厂入口与跨页选择逻辑。
- 回滚后端批量出厂接口与 OpenAPI `pack_factory_name` 处理逻辑。
- 回滚权限 SQL `49.sql`；如已授予权限，可同步移除对应 `ui_codes`。
