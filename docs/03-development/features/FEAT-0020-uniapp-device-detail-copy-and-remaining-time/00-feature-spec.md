# FEAT-0020 UniApp 设备详情页单体文案与充放电剩余时间调整 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-03-22
- related_feature: FEAT-0020
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - UniApp 设备详情页“参数设置 > 单体设置”中，部分参数文案未显式带出“单体”前缀，容易与总压参数混淆。
  - 仪表盘当前仍保留独立“剩余时间”区域，并将卡片展示为固定“充电时间”，与实际充/放电语义不一致。
- 目标：
  1. 单体设置分组内，所有未带“单体”的文案统一补齐“单体”前缀。
  2. 仪表盘移除顶部“剩余时间”区域。
  3. 仪表盘时间卡片改为“充放电剩余时间”，并根据当前状态动态显示“充电剩余时间”/“放电剩余时间”。
  4. 充电时展示正向分钟值，放电时展示负向分钟值，默认空闲状态显示通用标题和占位值。

## 2. 范围
### In Scope
- 调整 `fjbms-uniapp/pages/device-battery/components/params-tab.vue` 的单体分组文案展示。
- 调整 `fjbms-uniapp/pages/device-battery/components/dashboard-tab.vue` 的剩余时间展示结构。
- 更新 `fjbms-uniapp/lang/zh-CN.ts`、`fjbms-uniapp/lang/en-US.ts` 文案。
- 确认使用状态寄存器 `0x111`（放电剩余时间）与 `0x112`（充电剩余时间）作为展示数据源。

### Out of Scope
- 不改动后端协议。
- 不改动 Web 管理端设备详情页。
- 不新增独立 BLE 读指令轮询逻辑，继续复用现有整块状态寄存器读取。

## 3. 验收标准
1. 参数设置页“单体设置”中，诸如“过充告警电压”显示为“单体过充告警电压”。
2. 仪表盘顶部不再展示独立“剩余时间”区域。
3. 仪表盘时间卡片默认标题显示“充放电剩余时间”。
4. 充电中时标题显示“充电剩余时间”，数值显示 `100min` 这类正值格式。
5. 放电中时标题显示“放电剩余时间”，数值显示 `-100min` 这类负值格式。
6. 空闲状态下保留通用标题，数值显示 `-`。

## 4. 风险与约束
- 当前充/放电状态依赖 `indicatorStatus.charging/discharging` 判定，若固件状态位异常，标题可能回退为默认文案。
- 单体文案本次按“单体设置”分组展示层统一补前缀，避免影响其它分组或协议标签口径。

## 5. 回滚方案
- 回滚 `params-tab.vue` 中的单体前缀拼装逻辑。
- 回滚 `dashboard-tab.vue` 中的时间卡片动态标题与负值展示逻辑，恢复原有顶部剩余时间区域和固定“充电时间”卡片。
