# FEAT-0043 UniApp 头像展示与裁切优化 - 测试报告

- status: in_progress
- owner: payhon
- last_updated: 2026-04-17
- related_feature: FEAT-0043
- version: v0.1.0

## 1. 静态验证
- [x] `cd fjbms-uniapp && pnpm exec tsc --noEmit`

## 2. 手工验证清单
- [ ] “我的”页加载超大横图头像时，头像在圆形容器内居中展示。
- [ ] “我的”页加载超大竖图头像时，头像在圆形容器内居中展示。
- [ ] 进入“我的 > 设置”后，右侧头像与“我的”页当前头像一致。
- [ ] 无头像用户进入“我的 / 设置”时，两处均展示默认头像。
- [ ] 设置页从相册选择头像后，会先进入裁切界面，再上传保存。
- [ ] 设置页拍照后，会先进入裁切界面，再上传保存。
- [ ] Android 真机拍照返回后，APP 不闪退，且能正常进入裁切界面。
- [ ] Android 真机拍照返回后，直接使用 `chooseMedia` 返回图进入裁切，不再因二次 `compressImage` 闪退。
- [ ] 裁切时可通过拖拽、缩放、旋转调整头像主体方向与区域。
- [ ] 裁切界面支持双指缩放放大/缩小图片，并保持边界约束正常。
- [ ] 取消裁切后，不会修改用户当前头像。
- [ ] 裁切失败或上传失败时，旧头像保持不变，并显示对应提示。
- [ ] 头像保存成功后，“我的”页与设置页头像即时刷新。
- [ ] 重新登录后，头像仍正确显示。
- [ ] 微信小程序授权头像/昵称更新链路未受本次修改影响。

## 3. 回归重点
- [ ] 头像 URL 为云存储外链时，展示不回退为错误相对路径。
- [ ] 头像 URL 为历史内部路径时，仍可正常展示。
- [ ] 点击裁切确认按钮时，不再出现 `TypeError: uni.$u.t is not a function`。
- [ ] App 启动日志中不再出现 `getApp() failed` 告警。
