# FEAT-0043 UniApp 头像展示与裁切优化 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-17
- related_feature: FEAT-0043
- version: v0.1.0

## 1. 背景与目标
- 当前 `fjbms-uniapp` “我的”页头像直接用图片元素展示，面对超大横图、竖图时，头像区域缺少统一容器裁剪口径，运行态容易出现展示不稳定。
- “我的 > 设置”页右侧头像展示与“我的”页未统一复用，头像数据源、默认图和 URL 处理逻辑分散。
- 用户在设置头像时，当前是选图后直接上传，没有裁切步骤，无法按头像区域对焦人物主体。

本次目标：
1. 统一“我的”页与设置页头像展示能力，确保头像始终按容器居中展示。
2. 设置页右侧明确展示当前用户头像。
3. 将非微信授权头像修改流程改为“选图/拍照 -> 裁切 -> 上传保存”。

## 2. 范围
### In Scope
- `fjbms-uniapp/pages/my/my.vue`
- `fjbms-uniapp/pages/my/setting/index.vue`
- `fjbms-uniapp/common/avatar.ts`
- `fjbms-uniapp/components/app-avatar/`
- `fjbms-uniapp/lang/zh-CN.ts`
- `fjbms-uniapp/lang/en-US.ts`
- FEAT-0043 文档与 `docs/04-project-tracking/board.md`

### Out of Scope
- 不新增后端接口，不修改头像上传/资料保存接口结构。
- 不改动微信小程序 `u-wx-auth` 授权头像直传链路。
- 不调整后台用户资料或 Web 端头像展示逻辑。

## 3. 用户价值
- 用户在“我的”页和设置页看到的头像展示一致，不会因原图比例极端而错位。
- 设置页可即时确认当前头像，不需要来回切页核对。
- 用户可以在上传前裁切头像主体，减少头像成图不理想导致的重复上传。

## 4. 验收标准
1. “我的”页头像始终在圆形容器内等比填充并居中展示。
2. 设置页“修改头像”行右侧展示当前头像；无头像时显示默认头像。
3. 设置页从相册或拍照选择图片后，必须先进入 1:1 裁切界面，确认后才上传。
4. 裁切后头像上传仍走现有 `/api/v1/file/up` 与 `/api/v1/app/auth/profile`。
5. 头像展示兼容云存储 URL、历史内部路径和默认头像回退逻辑。
6. 微信小程序授权头像/昵称更新链路保持现状，不受本次裁切流程影响。

## 5. 风险与约束
- 头像 URL 解析必须兼容 FEAT-0038 已修复的云存储地址归一化，不能退回为简单拼接相对路径。
- `u-cropper` 为现有三方组件封装，优先复用其公开能力，避免修改内部实现带来跨平台回归。
- 裁切确认与上传保存为串行流程，上传失败时必须保留旧头像不变。

## 6. 回滚方案
- 回滚 UniApp 头像展示组件与设置页裁切上传流程。
- 恢复“修改头像”选图后直传逻辑。
- 保留 FEAT-0043 文档记录，并按实际状态更新看板。
