# FEAT-0025 APP 联系客服单页内容接入 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-03-25
- related_feature: FEAT-0025
- version: v0.1.0

## 1. 背景与目标
当前 `fjbms-uniapp` 的“我的 > 联系客服”仍是占位页，无法通过后台统一维护内容。需要参照“用户政策”“隐私政策”的现有实现，将“联系客服”纳入 APP 内容管理的单页内容体系，实现后台可编辑、可发布，移动端可展示。

## 2. 范围（In Scope / Out of Scope）
In Scope:
- 后端 APP 内容接口支持 `contact_service` 作为单页内容 `content_key`。
- 管理端 `APP 管理 > 内容管理 > 单页内容` 新增“联系客服”选项，并复用现有保存/发布流程。
- UniApp “我的 > 联系客服”页面改为读取并展示后台维护的单页内容。
- 同步接口文档与项目看板。

Out of Scope:
- 不新增新的数据库表结构。
- 不改动 FAQ、意见反馈提交流程。
- 不新增电话拨号、微信拉起等交互能力，本次仅提供富文本/Markdown 单页展示。

## 3. 用户价值
- 运营人员可在后台统一维护联系客服文案，避免客户端发版才能更新联系方式。
- 用户在移动端可看到最新客服渠道、服务时间或说明信息。
- 单页内容能力在“用户政策 / 隐私政策 / 联系客服”之间保持一致，降低维护成本。

## 4. 验收标准
- 管理端单页内容下拉中出现“联系客服”选项。
- 选择“联系客服”后可按应用、语言保存草稿并发布。
- UniApp “我的 > 联系客服”进入后展示已发布内容；未发布或无内容时展示空态。
- 接口文档和看板已同步更新。

## 5. 风险与约束
- 现有租户、应用与语言解析逻辑需保持兼容，不能影响已有 `user_policy`、`privacy_policy`、`about_us`。
- 单页内容仍沿用 Markdown 转 HTML 渲染，客服文案中若包含复杂嵌入内容，显示效果受 `rich-text` 支持范围约束。

## 6. 回滚方案
- 回退 `contact_service` 相关前后端改动，恢复 UniApp “联系客服”占位页。
- 管理端若已录入该类内容，保留数据库数据但不再提供入口。
- 回退 FEAT-0025 文档与看板条目。
