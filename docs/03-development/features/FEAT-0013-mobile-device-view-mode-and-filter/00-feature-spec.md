# FEAT-0013 移动端设备添加分流、首页视图切换与筛选增强 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-03-12
- related_feature: FEAT-0013
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - 当前移动端“扫码/蓝牙添加设备”统一复用终端用户绑定激活链路，机构用户添加设备时也会写入绑定关系并激活设备，不符合业务要求。
  - 当前首页“我的设备”仅展示当前账号绑定设备；机构用户虽已有机构设备页，但不满足“添加后也要在首页看到”的要求。
  - 当前首页筛选图标未接入真实筛选交互，且机构用户存在大量设备场景，终端用户也需要按名称/序列号/MAC 地址筛选。
- 目标：
  1. 按用户类型拆分移动端设备添加语义：
     - 终端用户：添加 = 绑定 + 激活。
     - 机构用户：添加 = 加入“我的设备”列表，不绑定、不激活。
  2. 机构用户支持在首页切换两种查看模式：
     - 我添加的设备
     - 终端用户绑定设备
  3. 首页“我的设备”筛选图标在终端用户和机构用户场景下均可用，并按模式提供对应筛选条件。

## 2. 范围
### In Scope
- 后端：移动端设备添加接口按 `user_kind` 分流处理。
- 后端：新增机构用户“我添加的设备”记录模型与查询接口能力。
- 后端：首页设备列表接口支持按 `view_mode` 返回不同数据集，并支持名称/序列号/MAC/添加日期等筛选。
- 后端：机构用户查看“终端用户绑定设备”时，仅可查看当前机构权限范围内的数据。
- 后端：修正激活日志口径，确保机构用户“仅添加”不会被误记为设备激活。
- UniApp：“我的”页面在设置图标旁新增视图切换图标，仅机构用户显示。
- UniApp：首页“我的设备”支持模式标签展示、筛选弹层、分页加载和模式切换后的数据刷新。
- UniApp：长按卡片菜单按模式区分动作：
  - 终端用户：重命名、解绑。
  - 机构用户“我添加的设备”：移除。
  - 机构用户“终端用户绑定设备”：只读，不提供解绑/移除。

### Out of Scope
- 不改动 WEB 管理端设备列表与终端用户管理页面。
- 不引入新的角色体系或新的组织权限模型。
- 不在本次需求中扩展“终端用户绑定设备”视图的额外筛选项（如手机号、绑定时间）；本次先满足名称/序列号/MAC 地址。
- 不在本次需求中调整设备详情页业务能力，只保证从不同首页视图进入详情页的基础可用性。

## 3. 用户价值
- 机构用户可以安全地批量扫描查看设备，不会误激活电池状态。
- 机构用户可以在首页快速切换“我添加的设备”和“终端用户绑定设备”，提升巡检、售后、参数查看效率。
- 终端用户可以在首页直接按名称/序列号/MAC 地址筛选自己的设备，减少查找成本。
- 激活统计、绑定关系和首页可见性语义分离，避免后续数据口径混乱。

## 4. 验收标准
1. 终端用户通过移动端扫码/蓝牙添加设备后：
   - 创建设备绑定关系；
   - 设备激活状态更新为已激活；
   - 首页“我的设备”可见该设备。
2. 机构用户通过移动端扫码/蓝牙添加设备后：
   - 不创建设备绑定关系；
   - 不更新设备激活状态；
   - 首页默认模式“我添加的设备”可见该设备。
3. 机构用户在“我的”页面点击设置旁切换图标后，可在首页切换查看：
   - 我添加的设备；
   - 当前机构权限范围内的终端用户绑定设备。
4. 首页筛选图标可点击并弹出筛选条件：
   - 终端用户：名称、序列号、MAC 地址。
   - 机构用户“我添加的设备”：添加日期、序列号、MAC 地址。
   - 机构用户“终端用户绑定设备”：名称、序列号、MAC 地址。
5. 机构用户“仅添加设备”操作不会进入激活日志，也不会影响激活统计口径。

## 5. 风险与约束
- 当前移动端首页依赖“绑定设备列表”缓存与逐条遥测查询；若直接沿用现状，会在机构用户大批量设备场景下产生性能问题，需要同步改为分页和服务端筛选。
- 当前激活日志把移动端 `provision/bind` 路径视为激活来源，若不修正，会把机构用户“仅添加”误算为激活。
- 首页标题仍叫“我的设备”时，机构用户在两种模式间切换可能产生理解歧义，需在 UI 上增加当前模式标识。
- 本次默认“序列号”对应现有 `device_number` 字段；若后续业务确认存在独立序列号字段，需要另行补充文档和实现。

## 6. 回滚方案
- 回滚 FEAT-0013 对应的移动端与后端改动，恢复当前单一路径：
  - 移动端添加统一走绑定激活；
  - 首页恢复仅显示绑定设备；
  - 机构用户继续使用独立“机构设备”页查看设备。
- 若已上线新增“机构添加记录表”，回滚时保留表结构但停止读写，避免影响历史数据审计。
