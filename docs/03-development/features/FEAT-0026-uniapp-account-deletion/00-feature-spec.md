# FEAT-0026 UniApp 账号注销 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-06-24
- related_feature: FEAT-0026
- version: v0.1.0

## 1. 背景与目标
当前 `fjbms-uniapp` 的“我的 > 设置”提供账号注销能力，非 PACK 场景要求用户输入当前密码进行二次验证。PACK 厂微信小程序隐藏了账号密码设置，微信登录用户无法获知密码，因此需要在 PACK 微信小程序环境下允许用户确认风险后直接注销账号。

## 2. 范围（In Scope / Out of Scope）
In Scope:
- UniApp 设置页在“账号绑定”下方新增“账号注销”入口。
- 点击注销前先展示风险确认；非 PACK 场景继续弹出密码输入框完成密码校验。
- PACK 厂微信小程序命中启用中的 PACK 小程序配置，且当前用户绑定该 AppID 的微信身份时，注销不要求输入密码。
- 后端新增 APP 端已登录用户注销接口，限制为终端用户（`END_USER`）自助调用。
- 注销时删除终端用户自身账号、登录身份以及移动端相关绑定/痕迹数据。
- 同步功能文档与项目看板。

Out of Scope:
- 不提供撤销注销、冷静期恢复或人工找回能力。
- 不改造后台 Web 端账号管理，不支持业务账号/租户账号自助注销。
- 不新增短信/邮箱二次验证流程；非 PACK 场景仍以当前密码校验为准。

## 3. 用户价值
- 用户可在移动端完成账号闭环管理，不必依赖人工客服处理基础销户诉求。
- 注销前明确提示数据删除风险，减少误操作。
- 密码校验可防止仅持有登录态的他人直接注销账号。
- PACK 微信小程序用户无需知道隐藏的账号密码，也能完成合规注销流程。

## 4. 验收标准
- “我的 > 设置 > 账号绑定”下方出现“账号注销”入口，视觉上与普通绑定项区分为高风险动作。
- 非 PACK 用户点击后先看到不可恢复提示，确认后需输入当前密码。
- PACK 厂微信小程序用户点击后只看到不可恢复提示，确认后直接发起注销。
- 后端仅在 `appid` 命中同租户启用中的 PACK 小程序配置，且当前用户存在 `${appid}:openid` 形式的 `WXMP_OPENID` 绑定时跳过密码校验。
- 密码错误时后端拒绝注销并提示失败。
- 密码正确时后端删除终端用户账号相关数据，客户端清空登录态并跳转登录页。
- 文档与 `docs/04-project-tracking/board.md` 已同步。

## 5. 风险与约束
- 注销是破坏性操作，必须避免对 `ORG_USER` 等业务账号开放。
- PACK 微信小程序免密注销不得只信任前端环境判断，必须由后端校验 PACK 配置和当前用户微信身份绑定。
- 终端用户可能绑定设备，删除账号时需同步清理 `device_user_bindings` 等移动端侧关联，避免残留脏数据。
- 当前系统登录态为 JWT，本次只做服务端数据删除与客户端本地 token 清理，不额外维护服务端 token 黑名单。
