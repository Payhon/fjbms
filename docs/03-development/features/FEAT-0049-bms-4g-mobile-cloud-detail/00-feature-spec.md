# FEAT-0049 BMS 4G 移动端云端详情链路 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-07-07
- related_feature: FEAT-0049
- version: v0.1.0

## 1. 背景与目标
- 4G BMS 设备通过 MQTT 主动上报数据，不需要也不能依赖移动端建立 BLE 连接。
- 当前移动端首页根据后台 `is_online` 可显示 4G 设备在线，但进入设备详情后首帧完整实时读取耗时约 6 秒，影响详情页可见速度。
- 目标是在 4G BMS 设备详情页先使用后台当前遥测与 `bms.snapshot` 快速填充首屏，再由 MQTT Socket 透传实时数据覆盖，核心数据目标在 2 秒内可见。
- 4G BMS 设备需要复用蓝牙 BMS 的 BOOT OTA 协议，通过 APP WebSocket 桥接 MQTT 透传完成升级。

## 2. 范围
### In Scope
- 后端新增 APP 电池当前遥测接口，复用绑定/组织设备权限校验。
- UniApp 设备详情识别 4G/4G+BLE 设备，优先建立 MQTT Socket 透传实时链路。
- UniApp 根据云端当前遥测合成仪表盘首屏展示数据，优先使用 `bms.snapshot`，实时透传返回后再覆盖。
- 电芯 Tab 在未收到 `cell.voltagesMv` 时展示明确空态。
- UniApp 设备详情在连接成功但首帧 BMS 实时状态未返回时展示加载等待态，避免仪表盘短暂显示空值。
- UniApp 参数设置基础分组首次展开时展示分组级加载态，读取完成后再展开列表。
- 后端注册 APP 端 MQTT Socket WebSocket 路由，供移动端 4G BMS 实时读取与 OTA 透传共用。
- 4G BMS OTA 使用与蓝牙 BMS 一致的 BOOT 升级协议，经 MQTT Socket 透传下发升级帧。

### Out of Scope
- 不在本期实现 4G 模块自身 OTA 任务下发；4G 模块升级包管理归属 FEAT-0051。
- 不在本期实现继电器控制等非 BMS OTA 的主动控制链路。
- 不改变 BLE-only 设备详情现有连接与轮询逻辑。

## 3. 验收标准
1. 4G-only 设备进入详情页后不出现 BLE 连接超时导致的离线状态。
2. 后台 `is_online=1` 或存在当前遥测时，移动端详情显示 MQTT 在线。
3. 无 `bms.snapshot` 时，仪表盘仍可展示 SOC、总压、电流、最高/最低电芯电压、压差等摘要数据。
4. 无 `cell.voltagesMv` 时，电芯 Tab 显示“未收到电芯明细数据”。
5. 从首页进入设备详情后，连接已建立但首帧 `readAllStatus()` 尚未返回时，仪表盘显示中英文加载提示，不显示空白/零值仪表数据。
6. 参数设置中的单体设置、总压设置、电流设置、温度设置首次点击时，右侧箭头切换为加载图标；数据读取完成后再展开并显示向上箭头。
7. 4G BMS 设备进入详情页后，先用当前遥测/快照显示核心数据，目标 2 秒内可见，实时透传返回后自动覆盖。
8. 4G BMS 设备 OTA 可从参数页沿用 BMS OTA 包检查与下载，经 `/api/v1/app/battery/socket/ws` 透传执行 BOOT 升级。
9. 4G/4G+BLE 设备通过 MQTT Socket 产生真实上行回包时，即使当前没有主动遥测上报，也应刷新设备在线状态并在详情页显示为 4G 在线。

## 4. 风险与约束
- 当前 4G 设备自动上报可能只包含摘要值，无法凭空还原单体电芯电压列表。
- 详情页首屏 2 秒目标依赖当前遥测接口与移动网络质量；真实耗时需以真机和线上接口响应为准。
- 4G OTA 经蜂窝网络和 MQTT 透传，需使用更长超时与发送节流，避免按 BLE 近场时延假设处理。
- 4G 在线状态不能仅依赖主动上报；但也不能仅因 APP 下发请求就置在线，必须以云端收到设备侧上行回包或主动遥测为依据。

## 5. 回滚方案
- 回滚 APP 当前遥测接口。
- 回滚 APP Socket WebSocket 路由注册。
- 回滚 UniApp 4G 首屏快照预填、Socket 透传 OTA 参数和 MQTT BOOT 帧匹配兼容逻辑。
