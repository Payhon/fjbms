# FEAT-0049 BMS 4G 移动端云端详情链路 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-30
- related_feature: FEAT-0049
- version: v0.1.0

## 1. 背景与目标
- 4G BMS 设备通过 MQTT 主动上报数据，不需要也不能依赖移动端建立 BLE 连接。
- 当前移动端首页根据后台 `is_online` 可显示 4G 设备在线，但进入设备详情后仍执行主动连接流程，失败后显示离线且面板无数据。
- 目标是在 4G-only 设备详情页使用后台当前遥测与 `bms.snapshot`，实现“有数据上报即在线”的移动端只读展示链路。

## 2. 范围
### In Scope
- 后端新增 APP 电池当前遥测接口，复用绑定/组织设备权限校验。
- UniApp 设备详情识别 4G-only 设备并跳过 BLE/MQTT 主动读链路。
- UniApp 根据云端当前遥测合成仪表盘展示数据，优先使用 `bms.snapshot`。
- 电芯 Tab 在未收到 `cell.voltagesMv` 时展示明确空态。
- UniApp 设备详情在连接成功但首帧 BMS 实时状态未返回时展示加载等待态，避免仪表盘短暂显示空值。
- UniApp 参数设置基础分组首次展开时展示分组级加载态，读取完成后再展开列表。

### Out of Scope
- 不在本期实现 4G 参数读写、OTA、继电器控制等主动命令链路。
- 不改变 BLE-only 设备详情现有连接与轮询逻辑。

## 3. 验收标准
1. 4G-only 设备进入详情页后不出现 BLE 连接超时导致的离线状态。
2. 后台 `is_online=1` 或存在当前遥测时，移动端详情显示 MQTT 在线。
3. 无 `bms.snapshot` 时，仪表盘仍可展示 SOC、总压、电流、最高/最低电芯电压、压差等摘要数据。
4. 无 `cell.voltagesMv` 时，电芯 Tab 显示“未收到电芯明细数据”。
5. 从首页进入设备详情后，连接已建立但首帧 `readAllStatus()` 尚未返回时，仪表盘显示中英文加载提示，不显示空白/零值仪表数据。
6. 参数设置中的单体设置、总压设置、电流设置、温度设置首次点击时，右侧箭头切换为加载图标；数据读取完成后再展开并显示向上箭头。

## 4. 风险与约束
- 当前 4G 设备自动上报可能只包含摘要值，无法凭空还原单体电芯电压列表。
- 云端详情链路仅负责只读展示；需要主动下发命令的功能必须另行设计命令通道。

## 5. 回滚方案
- 回滚 APP 当前遥测接口。
- 回滚 UniApp 4G-only 云端展示分支，恢复原主动连接逻辑。
