# FEAT-0010 机构菜单权限生效与首页分层改造 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-03-07
- related_feature: FEAT-0010
- version: v0.1.0

## 1. 背景与目标
- 背景：租户在“系统管理 > 权限管理”已为 PACK 厂家、经销商、门店、APP 用户配置菜单权限，但实际登录后仅显示首页，配置未生效。
- 目标：
  1) 修复机构类型菜单权限下发与生效链路，确保不同类型账号按配置展示菜单。
  2) 首页按用户类型分层展示，新增机构用户与终端用户两类首页内容。

## 2. 范围
### In Scope
- 后端：修复 `org_type_permissions` 与菜单返回接口联动；补齐 APP_USER 类型菜单生效逻辑。
- 后端：新增首页聚合数据接口，支持机构首页与终端用户首页。
- 前端：首页按用户类型渲染两类子组件（机构/终端用户），并接入对应数据。

### Out of Scope
- 不调整现有租户首页卡片配置编辑能力（Board 可视化编辑）。
- 不引入新的角色模型与权限中心。

## 3. 验收标准
1. 对指定租户分别配置 PACK 厂家、经销商、门店、APP 用户菜单权限后，相应账号登录菜单与配置一致。
2. 机构用户首页展示：
   - 当前机构范围电池总数
   - 电池状态统计
   - 最近 7 天与 30 天激活曲线
3. 终端用户首页展示：
   - 当前用户拥有电池总数
   - 欢迎信息
4. SYS_ADMIN/TENANT_ADMIN 维持原首页行为不受影响。

## 4. 风险与约束
- 历史租户可能存在旧权限数据（仅子菜单、不含父菜单），需兼容处理父级补全。
- 菜单权限与首页数据均受组织隔离约束，避免越权查询。

## 5. 回滚方案
- 回滚 FEAT-0010 对应前后端改动。
- 如需紧急止血，首页可回退到原租户首页渲染逻辑，菜单权限回退到原 Casbin 逻辑。
