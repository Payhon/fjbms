# FEAT-0032 UniApp 开发者模式 - 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-04-03
- related_feature: FEAT-0032
- version: v0.1.0

## 1. 背景与目标
- 背景：
  - `fjbms-uniapp` 当前缺少统一的“开发者模式”入口，排查线上或测试环境问题时，无法在登录页快速确认客户端版本与接口目标地址。
  - 后续内页还需要根据开发者模式做差异化展示，因此需要先建立一个可跨页面读取的全局状态。
- 目标：
  1. 在登录页通过连续点击顶部 LOGO 9 次开启开发者模式。
  2. 将开发者模式做成全局、可持久化状态，供后续内页直接复用。
  3. 在开发者模式下，于登录卡片底部展示基座版本号、应用版本号与当前 API BaseURL。

## 2. 范围
### In Scope
- `fjbms-uniapp/pages/login/login.vue` 新增 LOGO 连击触发逻辑与调试信息展示区。
- `fjbms-uniapp/store/` 新增开发者模式全局状态，并持久化到本地存储。
- `fjbms-uniapp/common/` 新增运行时调试信息读取工具，统一获取版本号与 BaseURL。
- `fjbms-uniapp/lang/zh-CN.ts`、`fjbms-uniapp/lang/en-US.ts` 新增开发者模式相关文案。
- 功能文档、测试记录、发布说明和看板同步回写。

### Out of Scope
- 本次不新增关闭开发者模式的 UI 入口。
- 本次不修改后端接口、认证流程或登录接口参数。
- 本次不在其他内页增加开发者模式差异化功能，仅建立全局开关能力。
- 本次不引入远程配置或灰度控制。

## 3. 用户价值
- 测试、实施与开发人员可以在登录页快速确认当前安装包的运行环境，降低排查版本错配和环境指向错误的成本。
- 后续内页若需要显示调试入口、协议信息、接口域名或额外诊断功能，可直接复用同一全局状态。

## 4. 验收标准
1. 在登录页顶部 LOGO 连续点击 9 次后，开发者模式被开启，并提示开启成功。
2. 开发者模式开启后，重新进入登录页或跳转其他页面时，状态仍可被全局读取，不因页面销毁而丢失。
3. 开发者模式下，登录卡片底部展示：
   - 基座版本号；
   - 应用版本号；
   - 当前后端接口 API BaseURL。
4. 非开发者模式下，上述调试信息不显示，原有登录流程不受影响。
5. 微信小程序、H5、APP 等非 APP 场景即使版本字段存在差异，也不会因读取调试信息导致页面报错。

## 5. 风险与约束
- UniApp 在不同平台上可获取的版本字段并不完全一致，必须增加兜底逻辑，避免空值或运行时报错。
- “连续点击”需要设置合理的重置窗口，避免用户分散点击时误触发。
- 开发者模式为全局状态，必须避免与现有登录态、租户配置等存储键冲突。

## 6. 回滚方案
- 回滚开发者模式 store、运行时调试信息工具和登录页相关展示改动。
- 回滚多语言字典新增文案。
- 回滚 FEAT-0032 文档和看板状态。
