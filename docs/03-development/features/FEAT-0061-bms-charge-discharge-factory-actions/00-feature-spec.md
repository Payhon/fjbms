# FEAT-0061 设备详情充放电控制工厂命令 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-06-25
- related_feature: FEAT-0061
- version: v0.1.0

## 1. 背景与目标
- 设备详情高级设置当前已有 `0x057A~0x057B` 工厂命令入口，但缺少针对充放电控制的快捷操作。
- 运维和工厂人员需要在 Web 与移动端设备详情中直接执行禁止充电、禁止放电、恢复允许充放电三项操作。
- 目标是在不新增接口、不修改协议封装的前提下，复用现有工厂命令写寄存器链路，并纳入设备参数权限配置。

## 2. 范围
### In Scope
- Web 设备详情 BMS 面板高级设置的工厂命令列表。
- UniApp 设备详情参数设置页高级参数弹层的出厂配置列表。
- 后台“权限管理 > 设备参数权限”的工厂命令权限树。
- FEAT-0061 文档与项目看板同步。

### Out of Scope
- 不新增后端 HTTP API。
- 不新增数据库字段或 SQL 迁移。
- 不自动给既有受限组织授予新权限。
- 不修改 `function:chargeAllowed` / `function:dischargeAllowed` 功能配置开关。

## 3. 用户价值
- 工厂和运维人员可在设备详情中快速执行充放电管控命令。
- 高风险按钮受机构类型设备参数权限控制，可按组织精细授权。
- Web 与移动端使用同一组权限 key 和命令字，减少配置口径差异。

## 4. 验收标准
1. 后台设备参数权限树“高级参数 > 出厂配置”展示 `禁止充电`、`禁止放电`、`允许充放电` 三个权限项。
2. 受限账号未勾选对应权限时，Web 与移动端均不显示对应按钮。
3. 勾选 `factory:disableCharge` 后，Web 与移动端显示 `禁止充电`，发送命令字 `0x80000000`。
4. 勾选 `factory:disableDischarge` 后，Web 与移动端显示 `禁止放电`，发送命令字 `0x40000000`。
5. 勾选 `factory:allowChargeDischarge` 后，Web 与移动端显示 `允许充放电`，发送命令字 `0x00000000`。
6. 三项操作均复用现有确认弹窗和离线/通道不可用拦截。

## 5. 风险与约束
- 三项为高风险设备控制命令，默认不通过 SQL 自动授予已有受限组织。
- `允许充放电` 通过写入 `0x00000000` 清空该命令字，必须保持与协议示例帧一致。
- 运行态验收依赖真实设备或通讯链路，本地静态校验只能覆盖映射、权限和类型正确性。

## 6. 回滚方案
- 回滚 Web 与 UniApp `FACTORY_ACTIONS` 中新增的三个工厂命令。
- 回滚中英文文案和后端设备参数权限树新增节点。
- 不涉及数据库迁移回滚。
