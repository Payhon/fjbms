# FEAT-0014 电池信息补全（电芯品牌/电池型号）- 功能规格

- status: in_progress
- owner: payhon
- last_updated: 2026-03-13
- related_feature: FEAT-0014
- version: v0.1.0

## 1. 背景与目标
- 背景：后台管理的电池列表当前缺少“电芯品牌”“电池型号”两个关键业务字段，PACK 厂家无法对已存在的电池档案做补录和统一维护。
- 目标：
  1) 在 BMS 管理 > 电池列表支持单条和批量“电池信息补全”。
  2) 电池列表支持按电芯品牌、电池型号筛选，并在列表中展示对应字段。
  3) 电池型号选择器支持在当前业务弹层内直接新增型号，无需跳转管理页。
  4) 信息补全操作需进入电池运营日志，便于追溯。

## 2. 范围
### In Scope
- 数据库为 `device_batteries` 增加电芯品牌、电池型号两个业务字段。
- 后端新增“电池信息补全”接口，并扩展电池列表/导出查询。
- 前端电池列表新增批量按钮、行操作、筛选项、展示列和信息补全弹窗。
- 新增独立电池型号选择组件，支持选择已有型号和原地新增型号。
- 电池详情页在 `BMS面板` 旁新增 `基本信息` 标签页，展示“新增 BMS”表单的只读信息。
- 电池列表操作菜单新增 `编辑 BMS 信息` 与 `删除`，并分别接入运营日志与删除确认。
- 权限系统新增两个按钮权限并补齐 PACK 厂家权限。
- 电池运营日志新增“信息补全”操作类型。

### Out of Scope
- 不改动 APP 端页面与接口。
- 不改动电芯品牌管理、电池型号管理页面的主体交互模式。
- 不新增独立“补全记录”页面，统一复用现有运营日志。

## 3. 用户价值
- PACK 厂家可在电池入库后补齐业务档案，提升后续追溯、筛选和统计能力。
- 运营人员无需跳转到电池型号管理页即可完成新增型号与信息补全，操作链路更短。
- 管理端可按品牌/型号快速筛选目标电池，提高批量处理效率。

## 4. 验收标准
1. 电池列表顶部支持多选后触发“电池信息补全”，对选中设备批量写入电芯品牌和电池型号。
2. 电池列表单条操作菜单的“生命周期”下包含“信息补全”入口。
3. 信息补全弹窗中可查看基础信息：序列号、MAC 地址、出厂日期、当前状态。
4. 电芯品牌下拉值使用 `seq_no`，显示 `name`；电池型号下拉值使用 `seq_no`，显示 `name`。
5. 电池型号选择器在允许创建模式下支持原地新增型号，新增成功后自动回填当前选择。
6. 电池列表查询条件新增“电芯品牌”“电池型号”筛选项，其中查询区域的电池型号选择器为只读创建模式。
7. 权限系统新增：
   - `bms_battery_list_batch_info_complete`
   - `bms_battery_list_action_lifecycle_info_complete`
8. 信息补全操作会写入电池运营日志，操作类型可被筛选和展示。
9. 电池详情页 `基本信息` 标签页可展示新增 BMS 时填写的只读信息：电池序列号ID、批号、产品规格、订单编号、BMS通讯类型、BMS型号、蓝牙Mac、4G通讯卡ID、出厂日期、质保到期、备注。
10. 电池列表操作菜单新增：
   - `编辑 BMS 信息`：复用新增 BMS 表单，支持编辑并写入运营日志。
   - `删除`：删除前弹出不可恢复确认，删除设备及关联业务数据并写入运营日志。
11. 电池列表 `BMS型号` 后新增 `产品规格` 列。
12. 电池列表第 1~2 列固定，剩余中间字段支持横向滚动。

## 5. 风险与约束
- `device_batteries` 采用保存 `seq_no` 而非主键 ID 的方式，运行时依赖租户维度基础数据表做关联展示，需保证品牌/型号序号在各自作用域内唯一。
- 电池型号为机构维度数据，需严格限制 PACK 厂家只能选择本机构型号；租户管理员新增型号时需明确归属 PACK 厂家。
- 历史数据可能存在未创建 `device_batteries` 扩展行的设备，信息补全接口需兼容自动补行或安全报错。

## 6. 回滚方案
- 回滚前端信息补全入口、筛选项与新组件接入。
- 回滚后端信息补全接口、列表关联查询和操作日志类型扩展。
- 回滚 SQL 补丁 `44.sql` 新增列与权限项；如已写入业务数据，需先导出备份后再执行列级回退。
