# FEAT-0012 BMS 历史数据查询与异步导出通知 - 功能规格

- status: review
- owner: payhon
- last_updated: 2026-03-09
- related_feature: FEAT-0012
- version: v0.1.0

## 1. 背景与目标
- 背景：当前 BMS 电池管理仅能查看有限实时/单项历史能力，缺少“按设备查看全部上报历史数据（遥测+属性）”的统一入口，也缺少可追踪的异步导出通知体验。
- 目标：
  1) 在 BMS「电池管理」下新增「历史数据」菜单与页面，支持蓝牙设备与 4G 设备统一查询。
  2) 页面采用“左设备筛选 + 右历史表格”布局，支持左侧折叠。
  3) 右侧支持日期范围筛选、长表/宽表切换、异步 Excel 导出。
  4) 导出完成通过 WebSocket 通知任务创建人，前端右上角消息图标闪烁提醒并支持下载。

## 2. 范围
### In Scope
- 后端：新增属性历史表、导出任务表、历史查询接口、异步导出任务、导出通知 WS、固定下载路由。
- 前端：新增 BMS 历史数据页面、长宽表切换、异步导出交互、全局消息图标与 Popover 列表下载。
- 菜单权限：新增 `bms_battery_history` 路由菜单与 `bms_battery_history_export` 按钮权限。

### Out of Scope
- 不做属性历史存量回填（仅从新表上线后开始累计）。
- 不改动旧版设备详情页内的既有历史弹窗逻辑。

## 3. 业务规则
1. 查询与导出时间范围均限制最大 31 天。
2. 数据范围固定为“遥测 + 属性历史”。
3. 导出内容跟随当前视图模式（`long` 或 `wide`）。
4. 通知对象仅任务创建人。
5. 下载成功后任务标记为已下载，并从“已完成未下载”列表移除。
6. 导出文件有效期 7 天，超时后不可下载。

## 4. 验收标准
1. 菜单可见：具备权限用户可在 BMS/电池管理下看到「历史数据」。
2. 左右布局：左侧设备筛选可折叠/展开；选中设备后右侧可查询。
3. 查询正确：长表/宽表均能返回分页数据，模式切换会重置分页并重新查询。
4. 异步导出：点击导出后返回任务号并提示“导出任务已生成”。
5. 通知链路：任务完成后前端收到 WS 消息，顶部消息图标闪烁。
6. 下载链路：通过固定路由 `GET /api/v1/battery/history/export/download?task_id=...` 成功下载并标记已下载。

## 5. 风险与约束
- 历史查询来源包含高频遥测，宽表在大范围查询时列数较多，需限制时间窗口与分页。
- WebSocket 连接在弱网下可能断开，前端需具备重连与初始列表兜底能力。
- 需避免跨租户/跨组织访问设备历史与导出任务。

## 6. 回滚方案
- 回滚前后端功能入口（路由、页面、API）。
- 停止导出任务创建，保留已生成文件按过期策略自然失效。
- 回滚 SQL 补丁 `42.sql` 中菜单权限增量（谨慎处理线上已分配权限编码）。
